1. Turborepo는 기존 패키지의 package.json 스크립트를 가져와서 실행 순서, 병렬 처리, 캐시를 관리하는 방식이다.

2. (기존 프로젝트 기준) 현재 구성되어 있는 npm workspace 구조를 거의 변경하지 않는다

3. 설정 파일이 turbo.json 하나 수준이라 팀원들의 러닝커브가 낮다

4. Vite, Vue, TypeScript ㄷㅇㅇ의 JS/TS 프로젝트와 결합이 잘된다.

5. 로컬/원격 캐시를 지원한다

6. Gitlab CI 에도 별도 종속 없이 붙이기 쉽다.

7. 기존에 사용하던 npm script를 그대로 활용한다.

# Nx 도입하지 않는 이유
1. 팀원 전체의 급격한 러닝커브 고려

2. Nx 의존도가 커짐

3. 현 프로젝트 대비 다소 과한 설정

4. 규모가 작은 경우, Nx의 기능으로 얻는 이득보다 설정/관리 비용이 더 크게 들어갈 수 있다.

5. 반순 빌드/캐싱만 원하는 경우 대비 상당히 무겁다
