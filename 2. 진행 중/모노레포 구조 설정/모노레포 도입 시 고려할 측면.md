### 관리 측면
- 코드 공유 : 서로 다른 프로젝트 사이 쉽게 소스 코드 공유
- 일관성 있는 도구 : 서로 다른 프로젝트들(심지어 서로 다른 프레임워크를 사용하더라도) 에서 일관된 개발 경험 제공
- 스케폴딩 : 새로운 프로젝트 초기 구성을 쉽게

### 속도 측면
- 로컬 캐싱 : 같은 머신에서 같은 것을 두 번 빌드/테스트 하지 않음
- 분산 캐싱 : 다양한 환경에서 캐시 아티펙트를 공유. 즉, 조직 단위로 여러 CI 환경에 겅쳐 같은 것을 두 번 빌드/테스트 하지 않음
- 로컬 작업 오케스트레이션 : 빌드 및 테스트 등의 작업을 순서에 맞게 병렬로 실행
- 분산 작업 실행 : 단일 시스템에서 실행되어 여러 시스템에 명령 전달
- 변화에 영향을 받는 프로젝트 감지 : 변경의 영향을 받을 수 있는 항목을 결정해 해당 프로젝트만 빌드/테스트

### 구조 파악 측면
- 워크스페이스 분석 : 추가 구성 없이 주어진 워크 스페이스의 의존성 관계를 분석
- 의존성 그래프 시각화 : 프로젝트 및 작업 간의 종속 관계를 시각화
