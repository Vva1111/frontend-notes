- build / dev / test / lint 를 루트에서 앱/패키지 대상으로 효율적으로 돌리는 기능
	- 작업 의존성 관리
	- 병렬 실행
	- 빌드 캐시
	- 변경된 앱/패키지만 빌드
	- CI 최적화

### Tool 비교
#### 1. Turborepo

Turborepo는 기존 패키지의 `package.json` 스크립트를 가져와서 **실행 순서, 병렬 처리, 캐시**를 관리하는 방식이다.

예를 들어:

```
{
  "scripts": {
    "build": "vite build",
    "lint": "eslint .",
    "type-check": "vue-tsc --noEmit"
  }
}
```

각 workspace에 이런 스크립트가 있으면 루트에서:

```
npx turbo run build
```

만 실행해도 패키지 의존 관계를 분석해 필요한 순서대로 실행한다. Turborepo는 패키지와 태스크의 관계를 DAG로 구성하며, 이전 결과가 유효하면 해당 작업을 캐시에서 복원한다.

### 장점

- 기존 npm workspace 구조를 거의 변경하지 않는다.
- 설정 파일이 `turbo.json` 하나 수준이라 이해하기 쉽다.
- Vue, Vite, TypeScript 같은 JS/TS 프로젝트에 자연스럽다.
- 로컬 캐시와 원격 캐시를 지원한다.
- GitLab CI에도 별도 종속 없이 붙이기 쉽다.
- 현재 사용하던 npm 스크립트를 그대로 활용한다.

원격 캐시를 사용하면 다른 개발자나 CI가 이미 수행한 동일 빌드 결과도 재사용할 수 있다.

### 단점

- Nx처럼 강력한 프로젝트 분석이나 코드 구조 규칙 강제 기능은 없다.
- `affected` 분석은 가능하지만 Nx만큼 세밀하고 기능이 풍부하지는 않다.
- 코드 생성기나 플러그인 중심의 개발 환경을 제공하지 않는다.
- 패키지 구조와 스크립트 설계는 개발팀이 직접 잘 잡아야 한다.

### 적합한 경우

```
apps/
  pcmp-frontend
  admin-frontend

packages/
  ui
  api
  utils
```

이런 구조에서 다음 정도가 목적이면 가장 적합하다.

- 변경되지 않은 공통 패키지 재빌드 방지
- 앱과 공통 패키지 빌드 순서 관리
- lint, test, type-check 병렬 실행
- CI 빌드 시간 단축

#### 2. Nx

Nx는 태스크 실행기라기보다 **모노레포 전체 개발 플랫폼**에 가깝다.

단순히 빌드 캐시만 제공하는 게 아니라 다음까지 다룬다.

- 프로젝트 의존성 그래프
- 변경 영향을 받은 프로젝트 분석
- 코드 생성기
- 라이브러리 경계 규칙
- 플러그인
- CI 분산 실행
- 프로젝트별 명령 관리

Nx는 `nx affected`를 사용해 두 Git 기준점 사이에서 실제 영향을 받은 프로젝트만 빌드하거나 테스트할 수 있다.

```
nx affected -t build \
  --base=origin/main \
  --head=HEAD
```

### 장점

- 프로젝트가 많아질수록 관리 능력이 뛰어나다.
- 어떤 앱이 어떤 패키지에 의존하는지 시각화하기 쉽다.
- 변경된 코드로 인해 영향을 받는 앱만 정확하게 실행할 수 있다.
- 프로젝트별 태스크 설정과 공통 설정을 체계적으로 관리할 수 있다.
- 모듈 간 참조 규칙을 강제할 수 있다.
- 캐시 입력과 출력 조건을 세밀하게 구성할 수 있다.

### 단점

- Turborepo보다 배워야 할 개념이 많다.
- 기존 npm scripts 중심 프로젝트에 넣으면 설정이 다소 과하게 느껴질 수 있다.
- Nx 플러그인이나 executor 중심으로 깊게 들어가면 Nx 의존도가 커진다.
- 작은 팀에서는 관리 기능보다 설정 비용이 더 크게 느껴질 수 있다.
- 단순히 빌드 캐시만 원하는 경우에는 상당히 무겁다.

### 적합한 경우

- 앱과 패키지가 수십 개 이상이다.
- 여러 팀이 하나의 모노레포를 사용한다.
- 앱별 소유권과 모듈 경계를 강제해야 한다.
- 변경된 앱만 정확하게 테스트·배포해야 한다.
- 공통 프로젝트 생성 규칙이 필요하다.
- CI 작업을 여러 runner에 분산할 계획이다.

#### 3. Lage

Lage는 Microsoft가 만든 JS 모노레포 태스크 러너다.

핵심은 Turborepo와 비슷하다.

- 태스크 병렬 실행
- 의존성에 따른 실행 순서
- 로컬 캐시
- 외부 스토리지를 이용한 캐시

공식 설명에서도 간단한 설정과 로컬 또는 자체 클라우드 스토리지 사용을 주요 특징으로 소개한다.

### 평가

성능이나 기능이 나쁜 건 아닌데, 지금 신규 도입 관점에서는 Turborepo보다 선택할 이유가 많지 않다.

- 자료와 사용자층이 상대적으로 작다.
- Turborepo와 역할이 상당 부분 겹친다.
- 팀원이 새로 들어왔을 때 친숙도가 떨어질 가능성이 크다.
- Vue/Vite 모노레포에서 특별한 이점을 주지는 않는다.

Microsoft 내부 도구와의 연계나 자체 캐시 인프라에 특별한 요구가 없다면 우선순위는 낮다.

#### 4. moon

moon은 Rust로 작성된 태스크 러너이자 모노레포 관리 도구다. JavaScript와 TypeScript뿐 아니라 Rust, Go, Ruby 등 여러 언어를 지원한다.

### 장점

- 실행 속도가 빠르다.
- 여러 언어가 섞인 저장소에 적합하다.
- 툴체인 버전까지 통합 관리할 수 있다.
- 태스크 상속과 공통 설정 기능이 있다.
- 자체 해싱 기반 캐시를 제공한다.

### 단점

- JS/TS만 사용하는 프로젝트에서는 기능이 과할 수 있다.
- Turborepo나 Nx보다 국내외 사용 사례와 자료가 적다.
- 설정 방식이 npm scripts 중심 흐름과 조금 다르다.
- 기존 개발자들이 별도의 moon 개념을 익혀야 한다.

백엔드 Go, Rust 프로젝트까지 같은 저장소에서 관리할 계획이 있을 때 의미가 커진다.
