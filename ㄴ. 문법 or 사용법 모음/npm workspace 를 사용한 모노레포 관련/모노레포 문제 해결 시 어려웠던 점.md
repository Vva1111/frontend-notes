- 의존성 문제
	- 공통 기능과 apps 내부의 개별 기능들을 어느것을 공통화 하여 packages 로 빼내고, apps 내부에서만 사용할 기능을 구분하는 것,
	- 공통 기능에서 사용할 타 모듈에 포함된 기능에 의존성을 얼마나 둘 것인가?
	- 예를 들어, 현재 구조에서 http 통신을 담당하는 모듈 같은 경우, 기본적인 default 기능은 공통 packages 내부에 존재하고 있고, apps 에서 해당 모듈을 import 한 후, 기능에 맞게 커스텀 한 모듈을 packages 의 공통 기능 모듈에 주입해서 apps 의 A 프로젝트가 커스텀한 내용을 그대로 사용할 수 있도록 구성
	- 이런 방식으로 i18n 언어 설정 파일도 구성하려 했으나 **순환 참조 문제**가 발생했고, Vue 의 shallowRef 로 해당 문제를 해결하긴 했으나, 공통 packages 에서 정의한 i18n의 json 파일의 의존성을 어떻게 해결할 것인가에 대한 고민이 남아있음
		- packages 내부의 공통 모듈, 컴포넌트 등에서 사용할 common locale json 이 있을 것이고, 이러한 common 다국어는 '확인' 등과 같은 진짜 common 한 것들이기 때문에 apps 내부의 기능들에서도 사용할 수 있다.
		- 모노레포 방식의 다국어 의존성 흐름은
			- default lang config 파일은 packages/ 에 공통 모듈로 존재
			- apps/ 내부에서는 해당 모듈을 import 해서 각 기능별, common 등 다국어 파일을 주입하고, 설정을 커스텀해 화면으로 보여줌.
