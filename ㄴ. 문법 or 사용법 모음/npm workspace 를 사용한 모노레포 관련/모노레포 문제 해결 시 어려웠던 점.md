### 의존성 문제
- **개별 기능들 중 어느 것을 공통 packages 로 빼내고, apps 내부에서만 사용할지 구분하고, 공통 모듈에서는 다른 공통 모듈에 대해 의존성을 얼마나 둘 것인가?

- 예를 들어, (현재 프로젝트 기준)http 통신을 담당하는 모듈의 경우, default(뼈대) 기능은 공통 packages 내부에 존재하고 있고, apps/A 에서 해당 모듈을 import 한 후, apps/A의 기능에 맞게 커스텀 한 모듈을 packages 의 공통 기능 모듈에 주입해서 apps/A가 커스텀한 내용을 그대로 사용할 수 있도록 구성

- 이런 방식으로 i18n 언어 설정 파일도 구성하려 했으나 **순환 참조 문제**가 발생, 공통 packages/ 에서 정의한 i18n의 json 파일에 대한 의존성 문제
	- packages 내부의 공통 모듈, 컴포넌트 등에서 사용할 common locale json 과 apps/A 네서 사용할 common locale json 이 별도로 각자 존재하는 상황
	
	- 현 프로젝트 모노레포 방식의 다국어 의존성 흐름은
		- default(뼈대) lang config 파일은 packages/ 에 공통 모듈로 존재
		- apps/ 내부에서 위 모듈을 import 해서 각 기능, common 등의 다국어 파일을 주입하고, 기타 추가 설정을 커스텀해 화면으로 보여줌.
			- 여기서 common json 파일을 어느곳에 둘 것인가의 문제가 발생함
			- common 이라는 건 apps 내부와 packages 관계 없이 모두 사용할 수 있는 단어들인데, 이걸 apps / packages 로 나눠서 사용하는건 효율적이지 않다고 생각해서 common json 을 packages 로 공통화 해서 사용 시도
			
			- 해당 방식으로 사용했을 때, 의존성 흐름은 아래와 같다
			1. apps/A 에서 packages/ 의 공통 lang config import
			2. apps/A 에서 packages/ 의 common lang json import
			3. apps/A 내부에서 langConfig 에 common json 주입
			4. packages 의 공통 모듈 기능은 apps/A가 커스텀한 langConfig 를 참조해야 하기 때문에 apps/A 에서 공통 모듈에 1에서 가져와서 A에 맞게 커스텀한 lang config 를 공통 모듈에 주입
			
			5. 여기서 문제가 생기는데, i18n의 lang 함수를 사용할 수 없는 문제가 발생
				- 초기 페이지 로드시 lang 함수가 번역된 텍스트를 반환하지 못하고 다국어 키값 자체를 화면에 노출시키는 문제 발생
				- vue-i18n 인스턴스가 완전히 설정, 준비되기 전에 lang 함수가 먼저 평가되어 버려 발생한 문제
				- 공통 기능 모듈을 모노레포의 packages로 분리하는 과정에서 앱과 패키지 간의 컴포넌트 렌더링 시점과 다국어 설정 시점이 미묘하게 엇갈려 런타임 에러 발생
				- **해결**
					1. **Props 에서의 지연 평가** : props 기본 값 설정  시 props 평가 시점에 바로 default: lang('common.~') 처럼 즉시 실행하는 방식이 아니라 default:()=>{t('common.~') 방식으로 함수 사용해서 컴포넌트가 인스턴스화되는 시점(i18n 초기화 완료 이후)까지 함수 실행을 늦춰 키값이 그대로 노출되는 현상 막음
					2. **computed 활용** : 사용자가 언어를 변경할 때 즉시 반응할 수 있도록 반응형 처리 computerd(())=> t('common.~')				
					3. **초기화된 t 함수 주입** : packages/ 내부의 공통 모듈이 자체적인 lang 인스턴스나 langConfig를 직접 import 해서 사용하면 의존성이 꼬이거나 초기화되지 않은 객체를 참조하는 문제 발생. 이를 해결하기 위해 실제 구동되는 앱(apps/A) 측에서 useI18n() 을 통해 이미 초기화가 끝난 t 함수 자체를 공통 모듈쪽으로 주입해 실행하도록 구조 변경
					
- 결론
	- 일정 문제로 기존 방식 그대로 가기로 함
