## 자기 소개

직관적인 정보, 자연스러운 서비스 흐름의 사용자 경험 위주 화면 설계, 개발과 DevOps 환경 구축, 개선을 통한 개발 생산성 향상까지 함께 고민합니다.

  

데이터 시각화 방면 **사용자 편의성 개선 경험**이 있습니다.

웹 페이지 최초 접근 시 최대 10초 정도의 과한 로드가 발생하는 현상을 발견하고, 사용자 경험에 악영향을 미칠 수 있다고 판단해 원인을 분석했습니다.

최초 페이지 진입점을 중점적으로 분석해 구글 폰트, 이미지 등을 다운로드하는 과정에 프로젝트에 사용하지 않는 다수의 파일까지 함께 로드하는 상황이 원인이라는 점을 알아낼 수 있었습니다.

문제 해결을 위해 사용하지 않는 파일 import 코드를 제거하고, 주로 사용하는 폰트와 이미지만을 import 해서 공통으로 사용할 수 있도록 프로젝트 내부 라이브러리화로 **초기 페이지 로딩 시간 약 2~3,000ms 감소**효과를 가져올 수 있었습니다.

  

또한, **개발 생산성 향상**을 위해 CI/CD 소요 시간의 약 70%를 감소시킨 경험이 있습니다.

프론트엔드, 백엔드, 배포를 담당하는 모든 코드가 포함된 레거시 프로젝트의 마이그레이션 과정에서 저장소에 코드를 한 번 저장,통합 하고 배포하기 위해서는 총 약 13분의 CI/CD 소요 시간이 필요했습니다.

약 13분의 CI/CD 소요 시간은 6명의 개발자가 코드를 적용시키고 배포된 결과물을 확인하기 위한 긴 대기시간을 필요로 하게 해 업무 병목을 일으키고, 코드를 한 번에 모아서 올리게끔 만들어 코드 병합 충돌을 다수 발생시켰습니다.

이러한 문제를 해결하기 위해

1. 각 코드를 프론트엔드 / 백엔드 / 배포 전용으로 관심사 분리

1. 프론트엔드 패키지 caching 작업으로 반복적인 패키지 다운로드 과정 제거

1. 빌드 도구(Vite) 설정을 통해 build 결과 assets의 크기 제한 및 경로 설정으로 파일 분리

1. CICD 수행 시 분기 처리를 통한 제한된 이미지 빌드

등을 통해 CI/CD 과정을 **10여 분에서 3분 대로 줄여 70%의 시간 감소 효과**를 얻어 업무 효율을 상승시킬 수 있었습니다.
