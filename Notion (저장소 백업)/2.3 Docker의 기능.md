## 1. Docker 이미지를 만드는 기능 (Build)

- Docker는 애플리케이션의 실행에 필요한 프로그램 본체, 라이브러리, 미들웨어, OS나 네트워크 설정 등을 하나로 모아 Docker 이미지를 만든다. (주로 Dockerfile 사용)

- Docker 이미지는 실행 환경에서 움직이는 컨테이너의 바탕이 된다.

- Docker 에서는 하나의 이미지에는 하나의 애플리케이션만 넣어 두고, 여러 개의 컨테이너를 조합해 서비스를 구축하는 방법을 권장한다.

- Docker 이미지는 애플리케이션의 실행에 필요한 파일들이 저장된 디렉토리라고 할 수 있다.

- 생성은 명령어를 사용해 수동으로 만들수도 있지만, Dockerfile이라는 설정 파일을 통해 자동으로 만들수도 있다. CICD와 인프라 구성 관리 등을 생각하면 Dockerfile을 사용해 관리하는 것이 바람직하다고 볼 수 있다.

- Docker 이미지는 변경 사항에 대해 레이어 관리를 하기 때문에 이미지를 겹쳐서 사용할 수 있다.

- 레이어 관리 - 추가 필요

  

## 2. Docker 이미지를 공유하는 기능 (Ship)

- Docker 이미지는 Docker 레지스트리에서 공유할 수 있다.

- Docker 공식 레지스트리인 Docker Hub 에서는 Ubuntu, CentOS와 같은 Linux 배포판의 기본 기능을 제공하는 베이스 이미지를 배포하고 있다. 이러한 베이스 이미지에 미들웨어나 라이브러리, 전개할 애플리케이션 등을 넣은 이미지를 겹쳐서 독자적인 이미지를 만들 수 있다.

- 개인이 만든 이미지를 Docker Hub에 올리는 것도 가능하다.

  

  

## 3. Docker 컨테이너를 작동시키는 기능 (Run)

- Docker는 Linux 상에서 컨테이너 단위로 서버 기능을 작동시키며, 이 컨테이너의 바탕이 되는 것이 Docker 이미지이다.

- Docker 이미지만 있으면 Docker 가 설치된 환경이라면 어디서든지 컨테이너를 작동시킬 수 있다.

- OS의 실행부터 시작하는 다른 가상화 기술과는 다르게 이미 움직이고 있는 OS 상에서 프로세스를 실행시키기는 것과 거의 똑같은 속도로 빨리 실행시킬 수 있다.

- Docker는 하나의 Linux 커널을 여러 개의 컨테이너에서 공유하고 있다.
    
    - 컨테이너 안에서 작동하는 프로세스를 하나의 그룹으로 관리하고, 그룹마다 각각 파일 시스템이나 호스트명, 네트워크 등을 할당하고 있다.
    
    - 그룹이 다르면 프로세스나 파일에 대한 엑세스를 할 수 없다.
    
    - 이러한 구조를 사용해 컨테이너를 독립된 공간으로서 관리하고 있다.
    
    - 제품 환경에서는 모든 Docker 컨테이너를 한 대의 호스트 머신(물리 머신) 에서 작동시키는 일은 드물며, 시스템 트래픽 증감, 가용성 요건, 신뢰도 요건 등을 고려한 후, 여러 대의 호스트 머신으로 된 분산 환경을 구축한다.
    
    - 보통 컨테이너 관리에 대해서는 오케스트레이션 툴을 이용하는 것이 일반적이다. 오케스트레이션 툴은 분산 환경에서 컨테이너를 가동시키기 위해 필요한 기능을 제공하고 있다.
    

  

## Docker Component

- Docker 를 구성하는 컴포넌트에 대한 설명

- Docker 는 핵심 기능인 Engine을 중심으로 컴포넌트를 조합해 애플리케이션 실행 환경을 구축한다.

  

1. Docker Engine : Docker 이미지 생성, 컨테이너 가동시키기 위한 Docker의 핵심기능 Docker 명령어 실행, Dockerfile에 의한 이미지 생성도 엔진에서 실행한다.

1. Docker Registry : Docker 이미지를 공개 및 공유하기 위한 레지스트리 기능. Docker Hub도 이 컴포넌트를 사용한다.

1. Docker Compose : 여러 개의 컨테이너 구성 정보를 코드로 정의하고, 명령을 실행함으로써 애플리케이션의 실행 환경을 구성하는 컨테이너들을 일원 관리하기 위한 툴

1. Docker Machine : 로컬 호스트용인 VirtualBox를 비롯해 AWS, EC2, Azure와 같은 클라우드 환경에 Docker의 실행 환경을 명령으로 자동 생성하기 위한 툴

1. Docker Swarm : 여러 Docker 호스트를 클러스터화하기 위한 툴. 클러스터를 관리하거나 API를 제공하는 역할은 Manager가, Docker 컨테이너를 실행하는 역할은 Node가 담당한다. 또한 오픈소스인 Kuberbetes도 이용할 수 있다.
