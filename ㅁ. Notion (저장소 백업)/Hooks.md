    가능하게 만들어준다.
    

- 함수형 컴포넌트는 기본적으로 렌더링이 될 때마다 함수 내부 코드가 모두 다시 실행

- 모든 코드가 다시 실행된다는 것은 기존에 가지고 있던 상태(state)를 전화 관리(기억)할 수 없다는 것이 된다. ( 렌더링 > 새롭게 선언 > 초기화 > 메모리 할당 - 반복 )

- Hook이 브라우저에 메모리를 할당해서 함수형 컴포넌트가 상태(state)를 가질 수 있게 함

- Hook을 만든 이유( 공식 홈페이지 )
    
    - 컴포넌트 사이에서 상태 로직 재사용의 어려움
    
    - 클래스형 컴포넌트 이해의 어려움
    
    - 클래스 자체 개념 ( this 등 )을 이해하기 어려움
    

- 최상위에서 호출, react 함수 내에서만 호출, use 사용 등의 Hooks 사용 주의점이 있다.

## useState

- 객체, 배열 데이터 자체를 수정하지 않고 복제본을 수정하는 것이 좋기 때문에 useState를 사용해 데이터 변경을 하는 것이 좋다

- 일반적인 지역변수를 ‘상태’로 업그레이드(?) 시킨 형태라고 생각하면 된다.

- ‘상태’ 는 React에서 중요한 개념 중 하나

- react 라이브러리에서 import해서 가져온 뒤 사용해야 한다.

  

- const [ aaa, setAaa ] = useState( 기본 값 ) 의 형태로 정의하며,
    
    변수에는 배열 값을,
    
    useState 함수의 인자인 기본 값은 해당 state가 제일 처음 가지는 값을 설정한다
    
    - aaa : 최초 설정 시 기본 값, 이후에는 변화되는 값을 저장한다
    
    - setAaa : 기본 값인 aaa를 변경할 수 있는 함수이며, setAaa와 같은 형식으로 사용
    
    - setAaa(변경하고자 하는 값); 으로 aaa의 값을 변경할 수 있으며,
        
        값의 변경이 생기면 재 렌더링이 된다.
        
    

  

- useState는 가지고 있던 기존 값과 변경된 값을 비교해 변화가 없다면 렌더링 하지 않는다.
    
    → 여기서 문제가 하나 발생하는데, 참조타입( 객체, 배열 데이터 등 )의 경우,
    
    자료구조 특성 상, 내부의 값이 변해도 해당 객체를 가리키는 메모리 주소 값은 변하지 않기 때문에 리액트는 값의 변화가 없다고 인식을 하게 되고, 객체 내부의 값을 바꿔도 렌더링이 일어나지 않게 된다.
    
    → 위의 문제를 해결하기 위해 객체 데이터 내부의 값을 변경하고, 렌더링 해서 적용하고 싶다면, 새로운 객체를 하나 만들고, 기존 객체 데이터를 복사해서 값을 추가한 뒤 새로운 객체 데이터를 만들어 메모리 주소 값(참조 값)을 변경시켜 리액트가 데이터 변화를 인식하고 렌더링을 할 수 있도록 만들어준다.
    
    - + [[JavaScript에서의 데이터 불변성]] 에 대해 추가 학습 필요
    

## useEffect

- useEffect( function, [] )
    
    - 기본적으로 렌더링 초기에 한번만 실행되는 함수를 설정하기 위해 사용한다
    
    - 두 번째 인자 값인 배열에 state를 넣으면 해당 state의 변경이 있을 때 마다 첫 번째 인자 함수 로직이 실행된다.
    

## useContext

- 필요한 props를 글로벌하게 사용할 수 있도록 도와주는 기능

- 하위 여러 단계를 거쳐 넘어가는 props의 경우, 중간에 불필요한 컴포넌트가 생길 수 있고, 유지보수 측면에서도 효율적이지 않다
    
    - 이러한 문제를 해결하기 위해 useContext를 사용해서 state를 전역적으로 관리한다
