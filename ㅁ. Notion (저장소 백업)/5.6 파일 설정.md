## **🌊 ADD 명령 (파일 추가)**

> [!important]
> 
> `ADD <호스트의 파일 경로> <Docker 이미지의 파일 경로>`
> 
> `ADD ["<호스트의 파일 경로>", "<Docker 이미지의 파일 경로>"]`

- 이미지에 호스트상의 파일이나 디렉토리를 추가
    
    - 호스트상의 파일, 디렉토리 혹은 원격 파일을 Docker 이미지 안으로 복사
    
    - 이미지 파일 경로 지정 시,
        
        - 파일(마지막 슬래시가 아님)일 때는 지정한 파일명을 추가
        
        - 디렉토리(마지막 슬래시)일 때는 URL로 지정한 것이 파일명이 됨
        
    
    - 호스트 파일이 `tar` 아카이브거나 압축 포맷일 경우 디렉토리로 압축 해제
    
    - 단, 원격 URL로 다운로드한 리소스는 압축이 풀리지 않음
    

- 쉘 형식과 Exec 형식 사용 가능

- 추가하고 싶은 호스트의 파일 경로에는 Dockerfile의 디렉토리 내부를 지정

- 호스트 파일 경로에는 Go 언어의 `filepath.Match` 함수와 일치하는 와일드카드 패턴 사용 가능

- 추가하고 싶은 파일이 원격 파일 URL인 경우, 퍼미션 600(사용자만 읽기/쓰기 가능)으로 설정됨
    
    - 원격 URL에서 파일을 다운로드할 수 있지만 보안 문제가 있음
    
    - 만약 취득한 원격 파일이 HTTP Last-Modified 헤더라면 추가된 파일의 `mtime` 값으로 사용
    

- 인증을 지원하지 않음
    
    - 원격 파일 다운로드에 인증이 필요할 경우 RUN 명령에서 `wget`이나 `curl` 명령을 사용
    

  

> [!important]
> 
> **호스트 파일 경로에서의** `**filepath.Match**` **룰**
> 
> - 특정 패턴과 파일이 일치하는지 확인하는 함수
> 
> - 절대 경로는 사용할 수 없음
> 
> - Docker 빌드 컨텍스트 보안 때문에 `../` 같은 상위 경로로도 접근 불가
> 
> - 홈 디렉토리(`~`)는 쉘에서만 동작하는 기능이라 사용 불가
> 
> - 재귀 검색(`**`)도 리눅스 쉘에서만 지원하므로 사용 불가
> 
>   
> 
> ```bash
> # [hos]로 시작하는 모든 파일 추가
> ADD hos* /docker_dir
> 
> # [hos]+임의의 한글자 룰에 해당하는 txt 파일 추가
> ADD hos?.txt /docker_dir
> ```

> [!important]
> 
> **와일드 카드**
> 
> |와일드카드|설명|
> |---|---|
> |`*`|전체|
> |`?`|한 글자|
> |`[ … ]`|대괄호 안에 있는 글자|

  

> [!important]
> 
> **HTTP Last-Modified 헤더**
> 
> ---
> 
> 마지막 갱신한 날짜(Last-Modified)의 타임스탬프

> [!important]
> 
> `RUN wget` / `RUN curl`
> 
> 두 명령 모두 mtime은 Last-Modified를 반영하지 않고 다운로드한 시간이 `mtime`이 됨. 아래 `mtime` 옵션을 사용하면 비슷한 효과를 누릴 수 있음.
> 
>   
> 
> **🔹** `RUN wget`
> 
> - 파일 다운로드 특화
> 
> - `RUN wget --user=<username> --password=<password> -O <저장 경로>`
> 
> - mtime 옵션: `--timestamping`
> 
>   
> 
> **🔹** `RUN curl`
> 
> - 다양한 HTTP 요청 가능
> 
> - `RUN curl -u <username>:<password> -o <저장 경로>`
> 
> - mtime 옵션: `--time-cond`

  

```docker
ADD myfile.txt /app/
```

- `myfile.txt`를 컨테이너 `/app/` 폴더에 복사

  

✅ **압축 파일 자동 해제**

```docker
ADD archive.tar.gz /app/
```

- `archive.tar.gz`가 자동으로 압축 해제됨

## **🌊 COPY 명령 (파일 복사)**

> [!important]
> 
> `COPY <호스트의 파일 경로> <Docker 이미지의 파일 경로>`
> 
> `COPY ["<호스트의 파일 경로>", "<Docker 이미지의 파일 경로>"]`

- 다운로드나 압축 해제 기능이 있는 ADD와 달리 호스트상의 파일을 이미지 안으로 ‘복사’만 진행

- 단순히 이미지 안에 파일 배치만 하고 싶은 경우 사용

  

> [!important]
> 
> **여러 개의 파일 복사**
> 
> ```docker
> COPY ["file1.txt", "file2.txt", "/app/"]
> ```
> 
> - `file1.txt`, `file2.txt`를 `/app/`에 복사

## **🌊 VOLUME 명령 (볼륨 마운트)**

> [!important]
> 
> `VOLUME <마운트 경로>`

- 이미지에 볼륨 할당

- JSON, 여러 인수로 된 문자열 지정 가능

- 컨테이너 내부의 특정 디렉토리를 볼륨으로 지정
    
    - Docker가 자동으로 데이터를 관리
    
    - 여러 디렉토리(`,`) 지정 가능
    

- 컨테이너가 삭제되어도 데이터 유지
    
    - 컨테이너는 영구 데이터 저장에 비적합  
        ➡ 영구 저장이 필요한 데이터는 컨테이너 밖의 스토리지에 저장
    
    - 영구 데이터는 Docker의 호스트 머신상의 볼륨에 마운트 하거나, 공유 스토리지를 볼륨으로 마운트하는 것이 적절함
    

  

```docker
VOLUME ["/var/log/"]
VOLUME ["/data", "/config"]
VOLUME /var/log /var/db
```

## **🌊 .dockerignore 파일 사용**

- Docker 이미지 빌드 시 **불필요한 파일 제외 가능**

- 불필요한 파일이 전송되지 않으므로 처리 속도가 빨라짐

- 여러 파일을 지정할 때는 줄 바꿈을 해서 파일명 나열

  

```bash
node_modules
*.git
*.log
```

- `node_modules`, `.git`, `.log` 파일들은 이미지에 포함되지 않음
    
    - 해당 파일을 사용(ADD 명령 등)할 경우, 파일이 존재하지 않아 오류 발생
