- ORM ( Object Relational Mapping )
    
    - 관계형 데이터베이스 맵핑
    
    - sql 쿼리문 없이 데이터베이스 핸들링
    
    - 데이터베이스의 테이블을 객체처럼 사용할 수 있도록 하는 기술
    
    - 효율 / 가독성 / 유지보수에 용이
        
        ( 기초 문법 참고 )

- SQL형 데이터베이스의 JOIN
    
    - 두 개 혹은 그 이상의 데이터베이스 테이블 사이에 공통점을 추출해 원하는 데이터를 새로 만들어 내는 방법
    
    - 서로 다른 테이블을 각각 조회해서 볼 수도 있지만, 데이터의 양이 방대하다면 핸들링이 감당할 수 없이 어려워진다
    
    - 테이블 JOIN을 위해서는 기본 키 ( Primary Key, PK ) / 외래 키 ( Foreign Key, FK ) 관계 형성이 필요 ( 일(PK) 대 다(FK) 관계 )
    
      
    
    - JOIN의 종류
        
        - INNER JOIN ( 내부 조인 ) : 지정한 두 테이블의 열에 모두 데이터가 있어야 한다, 기본적인 JOIN의 형태이며, 교집합 개념
        
        - OUTER JOIN ( 외부 조인 ) : 1개의 테이블에만 데이터가 있어도 결과를 출력
            
            ```sql
            SELECT <열 목록> FROM <첫 번째 테이블(LEFT TABLE)> *LEFT OUTER JOIN
            	<두 번째 테이블(RIGHT TABLE)> ON <조인 될 조건>
            
            
            * LEFT OUTER JOIN : 왼쪽 테이블에 해당하는 모든 값이 출력
            	RIGHT OUTER JOIN : 오른쪽 테이블에 해당하는 모든 값이 출력
            	FULL OUTER JOIN : 왼쪽과 오른쪽 테이블의 모든 값이 출력 
            										( 합집합 개념 )
            ```
            
        
          
        
        - CROSS JOIN ( 상호 조인 ) : 두 테이블의 모든 행과 행을 묶는다
            
            ※ 모든 행과 행을 묶기 때문에 두 테이블에 각각 3개의 행이 있다고 하면
            
            3 x 3 해서 9개의 결과를 출력
            
        
        - SELF JOIN (자체 조인 ) : 1개 테이블만 묶는다.
            
            ※ 1개의 테이블만 사용해 자체적으로 조인한다.
