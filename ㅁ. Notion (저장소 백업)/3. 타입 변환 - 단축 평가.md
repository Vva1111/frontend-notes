- 타입 변환의 경우, 개발자의 의도에 따른 타입 변환( 명시적 타입 변환 또는 타입 캐스팅 ) 개발자의 의도와는 상관 없는 자바스크립트 엔진에 의한 타입 변환( 암묵적 타입 변환 또는 타입 강제 변환 ) 의 두 가지 경우가 있다.

  

## 타입 변환

### 암묵적 타입 변환

![[Untitled 38.png]]

![[Untitled 39.png]]

  

- 기존의 원시 값(변경 불가능한 값) 을 변경하지 않고 다른 타입의 새로운 원시 값을 생성한다.

- 기존 변수 값을 재 할당해서 변경하는 것이 아니라, 피 연산자의 값을 암묵적 타입 변환해 새로운 타입의 값을 만들고 한 번 사용한 뒤 버리게 된다.

- 명시적 타입 변환과는 달리, 암묵적 타입 변환의 경우, 타입을 변환하겠다는 개발자의 의도가 코드에 명확히 나타나지 않는다.

- 이러한 타입의 변화를 예상하지 못하거나 예상이 일치하지 않는다면 오류를 발생할 가능성이 높아지는 코드가 된다.

  

### 암묵적 타입 변환의 종류

1. **문자열 타입으로 변환**
    
    - 1 + ‘2’ ⇒ ‘12’ (문자열)
        
        - + 연산자는 피연산자 중 하나 이상이 문자열이므로 문자열 연결 연산자로 동작한다.
        
        - 자바스크립트 엔진은 문자열 연결 연산자 표현식을 평가하기 위해 문자열 연결 연산자의 피연산자 중에서 문자열 타입이 아닌 피연산자를 문자열 타입으로 암묵적 타입 변환한다.
        
    

![[Untitled 40.png]]

  

![[Untitled 41.png]]

1. **숫자 타입으로 변환**
    
    - ‘ - , * , / ’ 산술 연산자의 역할은 숫자 값을 만드는 것이기 때문에 산술 연산자의 모든 피연산자는 코드 문맥 상 모두 숫자 타입이어야 한다.
    
    - 자바스크립트 엔진은 위의 이유로 산술 연산자 표현식을 평가하기 위해 산술 연산자의 피연산자 중에서 숫자가 아닌 타입을 숫자 타입으로 암묵적 변환한다.
    
    - 숫자 타입으로 변환할 수 없는 경우는 산술 연산을 수행할 수 없으므로 NaN가 나오게 된다.
    
    - 산술 연산자 뿐만 아니라, 비교 연산자( <, > 등 ) 또한, 크기를 비교해야 하므로 모든 타입은 숫자여야 한다.
    
    - + 단항 연산자 또한 피연산자의 타입이 숫자가 아니면 숫자로 암묵적 타입 변환을 수행한다.
    

![[Untitled 42.png]]

![[Untitled 43.png]]

> [!important] NaN : NaN의 경우, 컴퓨터가 표현하기 너무 크거나 작은 숫자, 숫자와 숫자가 아닌 값의 연산 결과 등으로 나오게 되는 값인데, 이러한 값은 ‘정의할 수 없는 수치’의 결과로써, 타입이 number로 평가되게 된다.

  

1. **Boolean 타입으로 변환**
    
    - 제어문, 반복문, 조건식과 같은 값들은 논리적 참/거짓으로 평가 되어야 하는 표현식들이기 때문에 조건식의 평가 결과를 Boolean 타입으로 암묵적 변환 한다.
    
    - 자바스크립트 엔진은 Boolean 타입이 아닌 값을 Truthy 혹은 falsy 값으로 구분한다. 즉, 값이 Boolean 타입으로 평가되어야 할 문맥에서 Boolean 타입이 아닌 값들을 Boolean 타입으로 암묵적 변환한다.
    
    - false, undefined, null, 0, -0, NaN, “” 등이 대표적으로 False로 변환되는 값들이다.

## 단축 평가

- 표현식을 평가하는 도중, 평가 결과가 확정 된 경우 나머지 평가 과정을 생략하는 작동 방식

- 논리 합( || ) 또는 논리 곱( && ) 의 경우, 평가 결과는 Boolean 값이 아닐 수 있다.

- 위 두 연산자 표현식은 언제나 2개의 피연산자 중 어느 한쪽으로 평가된다.

  

### && 연산자

![[Untitled 46.png]]

⇒ 두 개의 피연산자가 모두 true로 평가될 때, true를 반환한다.

![[Untitled 47.png]]

⇒ 좌항에서 우항으로 평가가 진행된다.

  

- 좌항에서 우항으로 평가하기 때문에, 두 번째 피연산자가 논리 곱 ( && ) 연산자 표현식의 결과를 결정하고, 이 때 논리 연산의 결과를 결정하는 두 번째 피연산자(문자열 ‘dog’)를 그대로 반환한다.

  

### || 연산자

![[Untitled 48.png]]

⇒ 두 개의 피연산자 중 하나만 true로 평가될 때, true를 반환한다.

![[Untitled 49.png]]

⇒ 첫 번째 피연산자가 true로 평가되었기 때문에 문자열 ‘cat’ 반환

  

- 논리 합 ( || )의 경우, 두 번째 피연산자까지 평가해 보기 않아도 표현식을 평가할 수 있기 때문에 첫 번째 피연산자 ‘cat’ 문자열을 그대로 반환한다.

  

- 위와 같이 논리 곱(&&), 논리 합( || ) 연산자는 논리 연산의 결과를 결정하는 피연산자를 타입 변환하지 않고 그대로 반환하며, 이를 **‘단축 평가’** 라고 한다.
