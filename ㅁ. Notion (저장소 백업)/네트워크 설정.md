### **3. google.com 과 172.25.80.120(개발서버) 라우트 확인**

→ gateway 가 200-100-1-1 / 200-100-1-1 동일하게 확인된다.  
→ 하지만 개발 서버로 ping을 날려보면 응답이 없다.

```bash
yepark@yeeun-Macmini kqc-common % route get google.com
```

### **4. sudo 명령을 사용하여 라우트 등록을 해준다.**

→ 개발 서버의 라우트 확인 후 ping을 날려보면 응답이 오는 것을 확인할 수 있다.

_==*reboot 시 등록이 풀리는 경우가 있어서 sudo명령을 다시 보내주면 된다!==_

```bash
yepark@yeeun-Macmini kqc-common % sudo route add 172.25.0.0/16 172.25.10.254
Password:
add net 172.25.0.0: gateway 172.25.10.254
yepark@yeeun-Macmini kqc-common % route get google.com

destination: default
       mask: default
    gateway: 200-100-1-1
```
