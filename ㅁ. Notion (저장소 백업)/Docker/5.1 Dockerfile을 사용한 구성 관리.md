# Dockerfile이란?

4장에서와 같이 명령을 통해 Docker를 사용하기 위해서는 다음과 같은 정보를 인프라 설계서나 파라미터 시트 등에 별도로 남겨 둘 필요가 있다.

- 베이스가 될 Docker 이미지

- Docker 컨테이너 안에서 수행한 명령

- 환경변수 등의 설정

- Docker 컨테이너 안에서 동작하는 데몬

  

Dockerfile은 위와 같은 Docker에서 작동시킬 컨테이너의 구성 정보를 기술하는 파일이다.

- **파일 형식:** 텍스트 (확장자 없음)

- **작성 방법:** 에디터에서 작성

- `**Dockerfile**` **이외의 파일명으로 작성한 경우**
    
    - Docker Hub에서 이미지 자동 생성 기능을 사용할 수 없음
    
    - 이미지를 빌드할 때 파일명을 명시적으로 지정해야 함

# Dockerfile의 기본 구문

```
명령 인수
```

- 관례적으로 대문자 사용 (대소문자 구분 없음)

  

## Dockerfile의 주요 명령

- `FROM`: 베이스 이미지 지정

- `RUN`: 명령 실행

- `CMD`: 컨테이너 실행 명령

- `LABEL`: 라벨 설정

- `EXPOSE`: 포트 익스포트

- `ENV`: 환경변수

- `ADD`: 파일/디렉토리 추가

- `COPY`: 파일 복사

- `ENTRYPOINT`: 컨테이너 실행 명령

- `VOLUME`: 볼륨 마운트

- `USER`: 사용자 지정

- `WORKDIR`: 작업 디렉토리

- `ARG`: Dockerfile 안의 변수

- `ONBUILD`: 빌드 완료 후 실행되는 명령

- `STOPSIGNAL`: 시스템 콜 시그널 설정

- `HEALTHCHECK`: 컨테이너의 헬스 체크

- `SHELL`: 기본 쉘 설정

  

## Dockerfile의 주석 서식

```bash
# 줄의 맨 앞에 '#'을 붙이면 그 줄 전체가 주석 처리됨
명령 인수 # 줄의 중간에 '#'을 붙이면 그 뒤부터 주석 처리됨
```

# Dockerfile 작성

> [!important]
> 
> **베이스 이미지**  
> Docker 컨테이너를 생성하는 기반이 되는 Docker 이미지

## FROM 명령 ==(required)==

베이스 이미지를 설정할 때 사용하는 명령

```docker
FROM <이미지명>
FROM <이미지명>[:태그명]
FROM <이미지명>[@다이제스트]
```

> [!important]
> 
> **다이제스트**  
> Docker Hub에 업로드 시 자동으로 부여되는 식별자
> 
> - 이미지를 고유하게 특정할 때 사용
> 
> - **다이제스트 확인 명령:** `docker image ls --digests <Docker Hub에서 취득한 이미지명>`  
>     
> 
> > **다이제스트 사용 유무 차이점**
> 
> - **사용 시:** 해당 이미지의 정확한 버전(불변)을 명시하게 되어 항상 동일한 이미지를 가져옴
> 
> - **미사용 시:** 사용하지 않고 태그만 지정하면 태그가 가리키는 이미지가 업데이트될 수 있으므로 빌드 시 예기치 않은 변경이 발생할 수 있음

  

### ==예)== CentOS의 버전 7을 베이스 이미지로 지정

```docker
FROM centos:centos7
```

- 태그명을 생략하면 최신 버전(latest)이 적용됨
