### Item 58. 모던 자바스크립트로 작성하기

- 아래 기능들은 모두 ECMAScript 2015 버전부터 도입된 것들이다.
    
    - ECMAScript 모듈( export / import ) 사용하기
    
    - 프로토타입 대신 클래스 사용하기
    
    ```tsx
    // 프로토타입 예제
    function Person(first, last) {
    	this.first = first;
    	this.last = last;
    }
    
    Person.prototype.getName = function(){
    	return this.first + ' ' + this.last;
    }
    
    const marie = new Person('Marie', 'Curie');
    const PersonName = marie.getName();
    ```
    
    ```tsx
    // 클래스 방식으로 변경
    class Person {
    	first: string;
    	last: string;
    	
    	constructor(first, last){
    		this.first = first;
    		this.last = last;
    	}
    	
    	getName(){
    		return this.first + ' ' + this.last;
    	}
    }
    
    const marie = new Person('Marie', 'Curie');
    const personName = marie.getName();
    ```
    
    - var 대신 let / const 사용하기
        
        - scope 문제를 피하기 위해 let / const 변수 선언 방식 사용
        
    
    ```tsx
    function foo(){
    	bar();
    	function bar(){
    		console.log('hello');
    	}
    		
    ```
    
    - foo 함수를 호출하면 bar 함수의 정의가 호이스팅 되어 가장 먼저 수행되기 때문에 bar 함수가 문제 없이 호출되고 hello가 출력된다. 호이스팅은 실행 순서를 예상하기 어렵게 만들고 직관적이지 않다. 대신 함수 표현식 const aaa = () ⇒ { … }; 을 사용해 호이스팅 문제를 피하는 것이 좋다.
    
    - for(;;) 대신 for-of 또는 배열 메서드 사용하기
        
        - for-of 루프는 코드가 짧고 인덱스 변수를 사용하지도 않기 때문에 실수를 줄일 수 있다.
        
        - 인덱스 변수가 필요한 경우 forEach 메서드를 사용하면 된다.
        
        - for-in 문법도 몇 가지 문제점이 있기 때문에 사용하지 않는 것이 좋다.
        
    
    - 함수 표현식보다 화살표 함수 사용하기
        
        - this 키워드는 일반적인 변수들과는 다른 스코프 규칙을 가지기 때문에 자바스크립트에서 가장 어려운 개념 중 하나이다.
        
        - 일반적으로 this가 클래스 인스턴스를 참조하는 것을 기대하지만 예상치 못한 결과가 나오는 경우가 있다.
        
        - 대신 화살표 함수를 사용하면 상위 스코프의 this를 유지할 수 있다.
        
        - 인라인 또는 콜백에서는 일반 함수보다 더욱 직관적이며, 코드도 간결해지기 때문에 가급적 화살표 함수를 사용하는 것이 좋다.
        
    
    - 단순 객체 표현과 구조 분해 할당 사용하기
        
        - 단순 객체 표현 : 변수와 객체 속성의 이름이 같다면, 객체의 속성을 제외하고 객체를 선언할 수 있다.
        
    
    - 함수의 매개변수 기본 값 사용하기
        
        - 자바스크립트에서 함수의 모든 매개변수는 선택적(생략 가능)이며, 매개변수를 지정하지 않는다면, 기본 값은 undefined 로 간주된다.
        
        - 매개변수 기본 값을 지정하면, 해당 매개변수가 선택적 매개변수라는 정보를 나타낼 수 있으며
        
        - 기본 값 기반으로 타입 추론이 가능하기 때문에 매개변수에 타입 구문이 추가되지 않아도 된다.
        
    
    - async / await 사용하기
    
    - 연관 배열에 객체 대신 Map과 Set 사용하기
    
    - 타입스크립트에 use strict 넣지 않기
        
        - 타입스크립트에서 수행되는 안전성 검사가 strict 모드보다 훨씬 더 엄격한 체크를 수행한다.
        
        - 타입스크립트 컴파일러가 생성하는 자바스크립트 코드에 use strict 키워드가 포함된다.

### Item 61. 의존성 관계에 따라 모듈 단위로 전환하기

- 점진적 마이그레이션 상황의 경우, 모듈 단위로 마이그레이션을 진행하는 것이 이상적이다.

- 의존성과 관련된 오류 없이 작업하려면, 다른 모듈에 의존하지 않는 최하단 모듈부터 작업을 시작해서 의존성의 최상단에 있는 모듈을 마지막으로 완성해야 한다.
    
    - 서드파티 라이브러리의 타입 정보를 가장 먼저 해결해야 한다. ( @types )
    
    - 외부 API의 타입 정보를 추가한다. API는 해당 모듈에 의존하지 않기 때문이다.
    

- 마이그레이션 할 때는 타입 정보만 추가하고, 리팩토링은 진행하지 않는다.

- 추가적인 오류들
    
    - 선언되지 않은 클래스 멤버
    
    - 타입이 바뀌는 값
    

  

  

  

### Item 62. 마이그레이션의 완성을 위해 nolmplicitAny 설정하기

- nolmplicitAny 설정은 상당히 엄격한 설정이며, strictNullChecks 같은 설정을 적용하지 않더라도 대부분의 타입 체크를 적용한 것으로 볼 수 있다.

- 최종적으로 가장 강력한 설정은 “strict” : true 이다.
