## Docker의 클라이언트 툴

Docker는 Linux 커널 기능을 사용하기 때문에 일반적으로 Linux 배포판에 설치하여 사용한다. 하지만 개발 환경에서 사용할 수 있도록 클라이언트 PC용 툴을 제공하고 있다.

  

### 1) Docker for Mac

- macOS 상에서 네이티브 애플리케이션으로 동작하는 툴

- 'xhyve'가 사용됨

- 설치: [https://store.docker.com/editions/community/docker-ce-desktop-mac](https://store.docker.com/editions/community/docker-ce-desktop-mac)

> 자세한 내용은 69~72 페이지 참고

  

### 2) Docker for Windows

- Windows 10 Pro, Windows 10 Enterprise(64bit)인 경우 사용 가능한 툴

- Microsoft가 제공하는 'Hyper-V'가 사용됨

- 설치 조건
    
    1. Windows 버전 확인: 10 Pro 또는 Enterprise(64bit)
    
    1. Hyper-V 활성화:  
        제어판 → 프로그램 및 기능 → Windows 기능 켜기/끄기 → Hyper-V 체크
    
    ![[스크린샷_2025-01-20_150810.png]]
    
    1. VirtualBox, VMWare Player와 같은 서드파티 제품 가상화 환경이 설치되어 있다면 삭제
    

- 설치: [https://store.docker.com/editions/community/docker-ce-desktop-windows](https://store.docker.com/editions/community/docker-ce-desktop-windows)

> 자세한 내용은 73~78 페이지 참고

  

### 3) Docker Toolbox

- Docker for Mac, Docker for Windows가 작동하지 않는 머신에 사용 가능한 툴

- Oracle이 제공하는 'VirtualBox'가 사용됨

- 설치: [https://github.com/docker-archive/toolbox/releases](https://github.com/docker-archive/toolbox/releases)

## Linux에 설치하기 (Ubuntu 17.10)

> 자세한 내용은 79~81 페이지 참고

> 다른 Linux 배포판, 클라우드 환경에서 설치하는 방법: [https://docs.docker.com/installation/#installation](https://docs.docker.com/installation/#installation)

### 설치 사전 준비

1. apt 업데이트

```bash
sudo apt-get update
```

- **APT(Advanced Packaging Tool):** Debian 계열 OS에서 작동하는 패키지 관리 시스템

1. HTTPS를 경유하여 리포지토리를 사용할 수 있도록 패키지 설치

```bash
sudo apt-get install -y \
apt-transport-https \
ca-certificates \
curl \
software-properties-common
```

1. Docker 공식 GPG 키 추가

```bash
curl -fsSL <https://download.docker.com/linux/ubuntu/gpg> | sudo apt-key add -
```

1. GPG 키 확인

```bash
sudo apt-key fingerprint 0EBFCD88
```

1. Docker 리포지토리 등록

```bash
sudo add-apt-repository \\
	"deb [arch=amd64] <https://download.docker.com/linux/ubuntu> \
	$(lsb_release -cs) \
	stable"
```

1. apt 업데이트

```bash
sudo apt-get update
```

  

### Docker 설치하기

```bash
sudo apt-get install docker-ce
```

## Docker에서 'Hello world' (`docker container run`)

설치한 Docker가 올바르게 작동하는지 확인하기 위해 Docker 컨테이너를 작성하고 콘솔상에 "Hello world"를 콘솔상에 표시해 보자.

```bash
docker container run <Docker 이미지명> <실행할 명령>
```

- `docker container run`: 컨테이너 작성 및 실행

- `<Docker 이미지명>`: 바탕이 되는 Docker 이미지

- `<실행할 명령>`: 컨테이너 안에서 실행할 명령

### 예) Ubuntu 이미지로 Docker 컨테이너 작성 및 실행후 "Hello world" 표시

```bash
docker container run ubuntu:latest /bin/echo 'Hello world'
```

- `ubuntu:latest`: Ubuntu 최신 버전의 이미지

- `/bin/echo 'Hello world'`: Linux의 echo 명령 실행

  

### 명령 실행 과정

1. Docker 컨테이너의 바탕이 되는 Ubuntu의 Docker 이미지가 로컬 환경에 있는지 확인
    
    - 없는 경우
        
        1. Docker 리포지토리에서 이미지 다운로드
        
        1. 다운로드 완료후 컨테이너가 시작되고 Linux의 echo 명령 실행
        
    
    - 있는 경우
        
        1. 로컬 환경에 있는 Docker 이미지를 바탕으로 Docker 컨테이너 시작
        
    

> `docker container run` 명령의 옵션은 p.109~p.119(4.2 Docker 컨테이너 생성/시작/정지) 참고

## Docker 버전 확인 (`docker version`)

`docker version` 명령을 통해 Docker의 버전, Go 언어의 버전, OS, 아키텍처 등을 확인할 수 있다.

![[83p.실행결과.png]]

Docker는 ==클라이언트/서버 아키텍처==를 채택하고 있어서 Docker 클라이언트와 Docker 서버가 ==Remote API==를 경유하여 연결되어 있으며, Docker 명령은 서버로 보내져 처리된다.

### IT 용어 설명

- **클라이언트/서버 아키텍처:** 네트워크를 통해 클라이언트와 서버가 상호작용하는 ==분산 컴퓨팅== 모델
    
    - 특징
        
        1. **분리된 역할**: 클라이언트는 사용자 인터페이스와 사용자 경험을 담당하고, 서버는 데이터 처리와 비즈니스 로직을 담당함
        
        1. **확장성**: 서버를 확장하여 더 많은 클라이언트 요청을 처리할 수 있음
        
        1. **보안**: 서버에서 데이터와 비즈니스 로직을 중앙 집중적으로 관리하여 보안을 강화할 수 있음
        
        1. **유지보수 용이성**: 서버 측에서 업데이트를 수행하면 모든 클라이언트에 적용됨
        
    

- **분산 컴퓨팅 (distributed computing):** 여러 컴퓨터 시스템이 네트워크를 통해 협력하여 작업을 수행하는 컴퓨팅 방식. 단일 시스템의 한계를 극복하고 더 높은 성능과 신뢰성을 제공하기 위해 사용됨
    
    - 특징 및 장점
        
        1. **병렬 처리**: 여러 컴퓨터가 동시에 작업을 수행하여 처리 속도를 높임
        
        1. **확장성**: 시스템에 더 많은 컴퓨터를 추가하여 성능을 향상시킬 수 있음
        
        1. **신뢰성**: 한 컴퓨터가 고장 나더라도 다른 컴퓨터가 작업을 계속 수행할 수 있어 시스템의 신뢰성을 높임
        
        1. **자원 공유**: 네트워크를 통해 컴퓨터 자원을 공유하여 효율적으로 사용할 수 있음
        
    
    - 예) 클라우드 컴퓨팅, 분산 데이터베이스, 분산 파일 시스템
    
    - 대표적인 프레임워크: Apache Hadoop, Apache Spark, Kubernetes
    

- **Remote API:** 네트워크를 통해 원격 시스템과 상호작용할 수 있도록 설계된 API(Application Programming Interface). 클라이언트와 서버 간의 통신을 가능하게 하며, 주로 웹 서비스에서 사용됨
    
    - 특징
        
        1. **네트워크 통신**: Remote API는 HTTP, HTTPS, WebSocket 등 네트워크 프로토콜을 사용하여 클라이언트와 서버 간의 데이터를 주고 받음
        
        1. **표준화된 인터페이스**: REST, SOAP, GraphQL 등 표준화된 인터페이스를 통해 다양한 클라이언트가 동일한 방식으로 서버와 상호작용할 수 있음
        
        1. **플랫폼 독립성**: 클라이언트와 서버가 서로 다른 플랫폼이나 언어로 구현되어 있어도 상호작용이 가능함
        
        1. **보안**: 인증, 인가, 암호화 등을 통해 데이터의 무결성과 기밀성을 보장함
        
    
    - 장점
        
        1. **확장성**: Remote API를 사용하면 서버를 확장하여 더 많은 클라이언트 요청을 처리할 수 있음
        
        1. **유연성**: 다양한 클라이언트(웹, 모바일, 데스크톱 등)에서 동일한 API를 사용할 수 있음
        
        1. **재사용성**: 동일한 API를 여러 애플리케이션에서 재사용할 수 있어 개발 효율성을 높임
        
        1. **유지보수 용이성**: 서버 측에서 API를 업데이트하면 모든 클라이언트에 적용됨
        
    
    - API와 Remote API 차이점
        
        1. **위치**:
            
            - **API**: 로컬 시스템 내에서 호출될 수 있음
            
            - **Remote API**: 네트워크를 통해 원격 시스템에서 호출됨
            
        
        1. **통신 방식**:
            
            - **API**: 함수 호출, 메서드 호출 등 로컬 호출 방식
            
            - **Remote API**: HTTP/HTTPS, WebSocket 등 네트워크 프로토콜을 사용한 원격 호출 방식
            
        
        1. **사용 목적**:
            
            - **API**: 애플리케이션 내부 또는 동일 시스템 내의 모듈 간 상호작용
            
            - **Remote API**: 분산 시스템 간의 상호작용, 클라이언트-서버 아키텍처에서의 데이터 교환
