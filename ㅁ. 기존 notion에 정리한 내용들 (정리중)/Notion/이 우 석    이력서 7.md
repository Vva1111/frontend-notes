## ==자기 소개==

사용자 경험 강화와 개발자 업무 편의성 증가를 함께 고민하는 프론트엔드 개발자입니다.

  

Vue 3 + TypeScript, Quasar, ECharts로 SaaS 매니지먼트 / GPU, 네트워크 관제·관리 등의 화면을 개발했습니다.

페이지 로드 시간을 약 2~3초 단축해 사용자 **편의성을 개선**하고**,** GitLab CI/CD, Docker, K8S를 활용한 배포 파이프라인 변경으로 코드 통합·빌드·배포 시간을 13분 에서 약 70% 가량 단축하여 3분대로 줄여 **개발 생산성을 향상시킨** 경험이 있습니다.

  

[ **AI 페어 프로그래밍을 통한 렌더링 병목 현상 해결 및 리팩토링** ]

실무 개발 과정에서 AI를 단순한 코드 생성기가 아닌, 구조 설계와 문제점 해결을 함께 논의하는 '페어 프로그래머'로 활용하며 개발 생산성을 높이고 있습니다.

대규모의 가상 머신 자원 정보를 테이블 컴포넌트에 바인딩하는 과정에서 심각한 브라우저 렌더링 지연 현상을 겪었던 경험이 있습니다. 원인을 찾기 위해 구현되어 있는 Vue 3의 상태 관리 로직과 회면 렌더링 방식 구성을 명확한 컨텍스트로 AI에게 제공하고, 기술적 조언을 구했습니다.

AI와의 문답을 통해 분석한 결과, Vue가 대용량 배열 내부 객체들의 깊은 반응성까지 모두 추적하면서 발생하는 재귀적 연산 오버헤드가 병목의 핵심 원인임을 확인했습니다. 이를 해결하기 위해 기존 `ref`를 최상위 수준의 반응성만 갱신하는 `shallowRef`로 대체하는 방향으로 상태 관리 함수를 리팩토링했습니다.

결과적으로 불필요한 반응성 추적 연산을 제거하여 테이블 렌더링 지연과 무한 루프 에러 리스크를 완벽히 해결할 수 있었습니다. 이를 통해 직접 해결 방법을 찾아보며 교차 검증하는데 소요될 디버깅 시간을 대폭 단축할 수 있었습니다.

이러한 실무형 프롬프트 엔지니어링 역량을 바탕으로, 개발을 직접 주도하되, 동료 개념의 AI 에이전트 사용으로 워크플로우에 유연하게 통합하여 개발 속도와 코드 품질을 동시에 끌어 올릴 수 있는 역량을 보유하고 있습니다.

  

[ **로딩 속도 개선을 통한 사용자 경험 강화** ]

페이지 최초 접근 시 최대 5초 정도의 로드가 발생하는 문제가 있었고, 파악 결과 폰트, 이미지 등 다운로드 과정에서 사용하지 않는 파일이 함께 로드되는 현상을 확인할 수 있었습니다.

로딩 속도 개선을 위해 import 코드를 제거하고, 주로 사용하는 폰트와 이미지만을 공통으로 사용할 수 있도록 라이브러리화하여 **초기 페이지 로딩 시간 약 2~3,000ms 감소** 효과를 얻을 수 있었습니다.

  

[ **배포 파이프라인 개선 및 GitLab 을 활용한 자동화 구성으로 개발 업무 편의성 증가** ]

프론트엔드, 백엔드, 배포 코드가 모두 포함된 레거시 프로젝트를 통합/배포하기 위해서는 총 약 13분의 소요 시간이 필요했습니다.

약 13분의 통합/배포 과정은 업무 병목과 코드 병합 충돌을 다수 발생시켜 생산성에 악영향을 미쳤고, 이 문제를 해결하기 위해 코드 관심사 분리, 패키지 Caching, Build assets 크기 제한 및 파일 분류, CI/CD 분기 처리를 통한 제한된 이미지 빌드로 통합/배포 과정을 **10여 분에서 3분 대로 줄여 70%의 시간 감소 효과**를 얻어 업무 효율을 상승시킬 수 있었습니다.

  

개발은 문제 해결의 과정이라고 생각합니다. 항상 문제 해결을 위한 개발을 하고 있으며, 이를 통해 더 나은 화면과 생산성 향상을 제공할 수 있는 개발자입니다.
