- 포워드 프록시 라고도 불린다.

- 서버와 클라이언트 사이에서 중계기로써 대리로 통신을 수행하는 행위

- proxy server : 클라이언트가 자신을 통해 다른 네트워크 서비스에 간접적으로
    
    접속(proxy)할 수 있게 해주는 컴퓨터 시스템이나 응용 프로그램을 가리킨다.
    
    → 클라이언트 - 서버 사이에서 응답/요청을 처리하는 중개인 역할을 하는 또 다른 서버
    
    → 프록시 서버 중 일부는 서버에 요청 된 내용들을 캐시를 이용해 저장 해 둔다.
    

- [[GATEWAY]]와 다른 점은 프록시 서버는 동일 프로토콜을 연결한다는 점이다.

  

- 프록시 서버의 동작 과정
    
    1. 클라이언트에서 프록시 서버로 전달할 요청을 보낸다
    
    1. 프록시 서버는 클라이언트로부터 전달 받은 요청을 서버에 요청한다.
    
    1. 서버는 요청에 맞는 결과 데이터를 프록시 서버에 전달한다
    
    1. 프록시 서버는 서버로부터 전달 받은 데이터를 클라이언트에 다시 전달한다
    

  

그렇다면, 프록시 서버를 사용하는 이유는?

- 프록시 사용 목적
    
    1. 캐시 데이터 사용
        
        - 서버에 요청 된 내용을 캐시로 저장 해 놓는 프록시 서버에 요청을 할 경우 캐시에 저장 된 내용을 다시 가져올 수 있기 때문에 굳이 서버에 재 접속이 필요하지 않다.
            
            → 웹 서버로부터 웹 페이지를 캐시로 저장하는데 흔히 사용된다.
            
            → 전송 시간, 외부 트래픽을 줄일 수 있다.
            
        
    
    1. 보안
        
        - 프록시 서버를 클라이언트와 실제 서버 사이에 배치하면 실제 서버의 IP를 숨길 수 있다.(익명 유지) 또한, 중간의 프록시 서버를 방화벽으로 사용하기도 한다.
        
    
    1. 접속 우회
        
        - 프록시 서버의 IP를 제한이 되지 않는 IP로 설정하면 서버에서 국가 혹은 해당 IP를 막아놓아도 우회 된 프록시 서버의 IP를 사용해 접속할 수 있다.
        
    
    1. 네트워크 서비스나 콘텐츠로의 접근 정책 ( 원치 않는 사이트 차단 등 )
    
    1. 사용률 기록 및 검사를 위해 사용
    
    1. IP 추적을 피하기 위해 사용
    
    1. 서버 데이터 전달에 앞서 콘텐츠 검사를 위해 사용
    
    1. 밖으로 나가는 데이터를 검사(데이터 유출 방지)를 위해 사용
    

  

※ 리버스 프록시 :

네트워크에서 클라이언트를 대신해 한 대 이상의 서버로부터 자원을 추출하는 프록시 서버의 일종, 추출 된 자원들이 마치 웹 서버 자체에서 기원한 것처럼 해당 클라이언트로 반환된다.

관련 클라이언트들을 위해 임의의 서버에 접속하는 중간 매개체인 포워드 프록시와는 반대로, 리버스 프록시는 관련 서버들을 위해 임의의 클라이언트가 해당 서버에 접속하는 중간 매개체이다.
