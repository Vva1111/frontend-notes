- DOM에 대해
    
    - DOM ( Document Object Model )
        
        → 문서 객체 모델, HTML / XML 문서의 프로그래밍 interface
        
        → 문서의 구조화 된 표현을 제공
        
        → 프로그래밍 언어가 DOM 구조에 접근할 수 있는 방법을 제공해
        
        문서 구조, 스타일, 내용 등을 변경할 수 있게 돕는다.
        
        → 구조화 된 nodes, property, methods 를 가지고 있는 objects 로 문서를 표현한다.
        
        → 웹 페이지를 스크립트 또는 프로그래밍 언어들에서 사용될 수 있게 연결 시켜준다.
        
        ( MDN )
        
    
    - 웹은 문서를 공유하기 위해 만들어졌다
    
    - 이미 만들어진 HTML 구조의 일부를 변경하기 위해 전체 구조를 재구축 하는 방식은 비효율적이므로, 이미 만들어진 구조의 변경이 필요한 부분만 일부를 수정해 비용을 줄이고 효율을 높이기 위해 DOM 적용
    
    - 결국, HTML을 동적으로, 보다 더 효율적으로 변경하기 위해 HTML Document를 자바스크립트가 이해할 수 있는 객체 형태로 모델링하여 자바스크립트에서 조작할 수 있도록 만든 interface가 DOM의 개념이다.
    
    ![[ㄷ. 학습/SQLite/이미지/Untitled.png]]
    
    > DOM은 프로그래밍 언어는 아니지만, DOM이 없다면 자바스크립트 언어는 웹 페이지 또는 XML 페이지 및 요소들과 관련 된 모델이나 개념들에 대한 정보를 가지지 못한다.
    
    > Document 내의 모든 element는 Document를 위한 Document Object Model의 한 부분이기 때문에, 이러한 요소들을 DOM과 자바스크립트와 같은 스크립팅 언어를 통해 접근, 조작할 수 있는 것이다.
    
    > MDN
    
    - API(web or XML page) = DOM + JS (Scripting Language)
    
    - DOM은 프로그래밍 언어와 독립적으로 디자인 됐기 때문에 문서의 구조적 표현은 단일API를 통해 이용 가능하다. (DOM의 구현은 어떤 언어에서든 가능하다)
    
    - DOM이 생겨나기 이전에도 브라우저는 자바스크립트 이전부터 HTML를 파싱 / 표현할 수 있는 내부 구조를 가지고 있었고, 그러한 내부 구조는 자바스크립트로 이루어 진 것이 아니다. 단지 HTML을 파싱하고 표현할 수 있는 내부 구조를 자바스크립트를 통해 조작할 수 있도록 하는 API를 제공하는 형식으로 만들어진 것이다.
    
    - 이후 웹 표준이 제정되며 어떠한 언어로든 위의 구조에 접슨할 수 있는 API 명세를 통일할 필요가 생겼고, 그로 인해 DOM을 정의하게 된다.
    
    - 간단히 말하자면, DOM은 ‘HTML을 수정하는 방법을 컴퓨터가 알아듣게 작성하는 방법’ 이라고 생각하면 된다.
