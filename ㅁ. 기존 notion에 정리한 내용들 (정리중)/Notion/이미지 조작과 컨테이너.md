## **이미지 다운로드(docker image pull)**

```docker
docker image pull [옵션] <이미지명> [:태그명]
```

- 사용 예시
    
    ```docker
    # CentOS의 버전7 이미지 취득
    docker image pull CentOs:7
    
    # CentOS의 모든 태그 이미지 취득 (태그명 생략시 latest)
    docker image pull -a CentOs
    
    # TensorFlow의 URL을 지정하여 이미지 취득
    docker image pull gcr.io.tensorflow/tensorflow
    ```
    
    `태그명 생략` : 최신판(latest) 취득
    
    `-a` : 모든 태그 취득 ==→ 이미지명 지정 불가!==
    
    `--platform` : 다운로드 이미지 플랫폼 지정
    
    `-q` : 다운로드 진행상태 최소화, 다운로드 완료된 이미지만 출력
    
    `--no-cache` : 캐시된 이미지가 아닌 새로운 이미지 다운로드
    

  

## **이미지 목록 표시(docker image ls)**

취득한 이미지 목록 표시

```docker
docker image ls [옵션] [리포지토리명]
```

|옵션|설명|
|---|---|
|`-all`, `-a`|모든 이미지 표시|
|`--digests`|다이제스트 표시 여부|
|`--no-trunc`|결과를 모두 표시|
|`-quite`, `-q`|Docker 이미지 ID만 표시|

- 💡 Docker digest?
    
    Docker 이미지의 고유한 식별자  
    이미지가 변경되거나 업데이트될 때마다 새로운 digest 값이 생성된다.  
    이미지의 무결성을 보장하고, 이미지를 정확하게 참조하는 데 사용된다.
    

- 사용 예시
    
    ![[image 44.png]]
    
    docker image ls 실행 결과
    
    ![[image 45.png]]
    
    docker image ls —digest 실행 결과
    
    |항목|설명|
    |---|---|
    |REPOSITORY|이미지 이름|
    |TAG|이미지 태그명|
    |IMAGE ID|이미지 ID _==(이미지 작성시 랜덤한 문자열 부여)==_|
    |CREATED|작성일|
    |SIZE|이미지 크기|
    

> [!important]
> 
> **이미지 서명 및 검증(Docker Content Trust)**
> 
> 이미지의 정당성을 확인하기 위하여 서명 및 검증을 할 수 있다.
> 
> - 서명 : docker push 전 로컬에서 비밀키(Offline Key)를 사용하여 서명한다.
> 
> - 검증 : docker pull 시, image 작성자의 공개키(Taggiing Key)를 사용하여 검증한다.
> 
> **DTC 기능**
> 
> 유효화 `export DOCKER_CONTENT_TRUST=1` (유효화 설정시 pull 할 때 이미지 검증)
> 
> 무효화 `export DOCKER_CONTENT_TRUST=0`

## **이미지 삭제(docker image rm)**

|---|---|
|`--force` , `-f`|이미지 강제 삭제 (중간이미지도 함께 삭제)|
|`--no-prune`|중간 이미지를 삭제하지 않음|

**사용하지 않는 Docker 이미지는 주기적으로 정리**

```docker
# 중간 이미지 (태그 없는 <none> <none>) 삭제
docker image prune

# 컨테이너로 실행되지 않은 모든 이미지 삭제
docker image prune -a
# --dry-run : 가상 실행
docker image prune -a --dry-run
```

|옵션|설명|
|---|---|
|`--all` , `-a`|사용하지 않은 이미지(컨테이너로 실행되지 않은) 전부 삭제|
|`--force` , `-f`|이미지 강제 삭제|

# 4.2 Docker 컨테이너 생성/시작/정지

## **Docker 컨테이너의 라이프 사이클**

![[image 55.png]]

컨테이너 상태

|명령|설명|
|---|---|
|`docker container create`|**컨테이너 생성**  <br>• 이미지로부터 컨테이너를 생성  <br>• 이미지의 실체는 Docker에서 서버 기능을 동작시키기 위해 필요한 디렉토리 및 파일들  <br>• 명령을 실행하면 이미지에 포함될 디렉토리와 파일들의 스냅샷을 취함  <br>• create 명령은 컨테이너를 작성하기만 할뿐 컨테이너를 시작하지는 않음|
|`docker container run`|**컨테이너 생성 및 실행**  <br>• 이미지로부터 컨테이너를 생성하고, 컨테이너 상에서 임의의 프로세스를 시작  <br>• 포트 번호와 같은 네트워크도 설정함으로써 외부에서 컨테이너의 프로세스에 액세스 할 수 있음|
|`docker container start`|**컨테이너 시작**  <br>• 정지 중인 컨테이너를 시작할 때 사용  <br>• 컨테이너에 할당된 컨테이너 식별자를 지정하여 컨테이너 시작|
|`docker container stop`|**컨테이너 정지**  <br>• 실행 중인 컨테이너를 정지할 때 사용  <br>• 컨테이너에 할당된 컨테이너 식별자를 지정하여 컨테이너 정지  <br>• 컨테이너 재시작 `docker container restart`|
|`docker container rm`|**컨테이너 삭제**  <br>• 컨테이너 상태 확인 `docker container ps`  <br>• 컨테이너 일시 정지 `docker container pause`|

  

## **컨테이너 생성 및 시작(docker container run)**

```docker
docker container run [옵션] <이미지명>[:태그명] [인수]
```

|옵션|설명|
|---|---|
|`--attach` , `-a`|표준 입력, 표준 출력, 표준 오류 출력에 attach|
|`--cidfile`|컨테이너 ID를 파일로 출력|
|`--detach` , `-d`|컨테이너를 생성하고 백그라운드에서 실행|
|`--interactive` , `-i`|컨테이너의 표준 입력 열기|
|`--tty` , `-t`|단말기 디바이스 사용|

- 사용 예시
    
    ![[image 56.png]]
    
    - `docker container run` 컨테이너 생성 및 실행
    
    - `-it` 콘솔에 결과 출력하는 옵션 → _==터미널에서 명령을 입력하고, 결과를 화면에서 볼 수 있는 인터랙티브한 환경==_
    
    - `--name "test1"` ==컨테이너명 →== _==임의의 이름으로 지정, 미지정시 랜덤으로 자동 설정됨==_
    
    - `centos` 이미지명
    
    - `/bin/cal` 컨테이너에서 실행할 명령 → _==/bin/cal (달력을 콘솔에 표시하는 명령)==_
    

  

## **컨테이너의 백그라운드 실행(docker container run)**

```docker
docker container run [실행 옵션] <이미지명>[:태그명] [인수]
```

|옵션|설명|
|---|---|
|`--detach` , `-d`|컨테이너를 생성하고 백그라운드에서 실행|
|`--user` , `-u`|사용자명 지정|
|`--restart=[no \| on-failure \| on-failure:횟수n \| always \| unless-stopped]`|명령의 실행 결과에 따라 재시작을 하는 옵션==`no` 재시작X==  <br>==`on-failure` 종료 status가 0이 아닐 때 재시작==  <br>==`on-failure:횟수n` 종료 status가 0이 아닐 때 n 번 재시작==  <br>==`always` 항상 재시작==  <br>==`unless-stopped` 최근 컨테이너가 정지상태가 아니라면 재시작==|
|`--rm`|명령 실행 완료 후 컨테이너 자동 삭제|

- 사용 예시
    
    ![[image 57.png]]

    - `-d` 백그라운드에서 실행(detach 모드)하는 옵션

## **컨테이너의 네트워크 설정(docker container run)**

```docker
docker container run [네트워크 옵션] <이미지명>[:태그명] [인수]
```

컨테이너 시작 시 네트워크에 관한 설정을 할 수 있다.

|옵션|설명|
|---|---|
|`--add-host=[호스트명:IP주소]`|컨테이너의 /etc/hosts에 호스트명과 IP주소 정의|
|`--dns=[IP주소]`|컨테이너용 DNS 서버의 IP 주소 지정|
|`--expose`|지정한 범위의 포트번호 할당|
|`--mac-address=[MAC 주소]`|컨테이너 MAC 주소 지정|
|`--net=[bridge \| container:<name\|id> \| host \| NETWORK]`|컨테이너 네트워크 지정|
|`--hostname` , `-h`|컨테이너 자신의 호스트명 지정|
|`--publish` , `-p[호스트 포트번호]:[컨테이너 포트번호]`|호스트와 컨테이너 포트 매핑|
|`--publish-all` , `-P`|호스트와 임의의 포트를 컨테이너에 할당|

Docker는 기본적으로 bridge network를 사용하여 컨테이너를 호스트 OS와 연결하지만, `--network` , `--net` 옵션을 통해 네트워크 설정을 제어할 수 있다.

```docker
docker network create [옵션] <네트워크 이름>
```

|설정값|설명|
|---|---|
|bridge|브리지 연결(기본값) 사용|
|none|네트워크에 연결하지 않음|
|container:[name \| id]|다른 컨테이너의 네트워크 사용|
|host|컨테이너가 호스트OS 네트워크 사용|
|NETWORK|사용자 정의 네트워크 사용|

## **자원을 지정하여 컨테이너 생성 및 실행(docker container run)**

```docker
docker container run [자원옵션] <이미지명[:태그명] [인수]

# CPU 시간의 상대 비율과 메모리 사용량 지정
docker container run --cpu-shares=512 --memory=1g centos

# 호스트 OS와 컨테이너 안의 디렉토리 공유
docker container run -v /Users/asa/webap:/usr/share/nginx/html nginx
```

|옵션|설명|
|---|---|
|`--cpu-shares` , `-c`|CPU의 사용 배분(비율)|
|`--memory` , `-m`|사용할 메모리 제한하여 실행 (단위는 b, k, m, g 중 하나)|
|`--volume=[호스트 디렉토리]:[컨테이너 디렉토리]` , `-v`|호스트와 컨테이너의 디렉토리 공유|

  

## **컨테이너를 생성 및 시작하는 환경을 지정(docker container run)**

```docker
docker container run [환경설정 옵션] <이미지명>[:태그명] [인수]

# 컨테이너 시작 시 환경변수 설정
docker container run -it -e foo=bar centos /bin/bash

# 환경변수 정의 파일로부터 일괄 설정
cat env_list
hoge=fuga
foo=bar

docker container run -it --env-file=env_list centos /bin/bash

# 컨테이너 작업 디렉토리 지정 실행
docker container run -it -w=/tensorflow centos /bin/bash
```

|옵션|설명|
|---|---|
|`--env=[환경변수]` , `-e`|환경변수 설정|
|`--env-file=[파일명]`|환경변수를 파일로부터 설정|
|`--read-only=[true \| false]`|컨테이너 파일 시스템 읽기전용 설정|
|`--workdir=[패스]` , `-w`|컨테이너 작업 디렉토리 지정|
|`-u` , `-user=[사용자명]`|사용자명 또는 UID 지정|

  

## **가동 컨테이너 목록 표시(docker container ls)**

```docker
# 현재 가동중인 컨테이너 상태 목록으로 표시
docker container ls [옵션]

# 컨테이너 명이 test1인 조건으로 필터링
docker container ls -a -f name=test1

# STATUS에 종료 코드가 0인 조건으로 필터링
docker container ls -a -f exited=0

# 컨테이너 ID와 가동 여부 상태를 콜론으로 구분하여 표시
docker container ls -a --format "{{.Names}} : {{.Status}}"

# 컨테이너 목록을 표 형식으로 출력
docker container ls -a --format "table {{.Names}} \t {{.Status}} \t {{.Mounts}}"
```

- 설정 옵션 및 출력 항목 설명
    
    |옵션|설명|
    |---|---|
    |`--all` , `-a`|실행중, 정지중을 포함한 모든 컨테이너 표시|
    |`--filter` , `-f`|표시할 컨테이너 필터링|
    |`--format`|표시 포맷 지정|
    |`--last` , `-n`|마지막으로 실행된 n건의 컨테이너만 표시|
    |`--latest` , `-l`|마지막으로 실행된 컨테이너만 표시|
    |`--no-trunc`|정보를 생략하지 않고 표시|
    |`--quiet` , `-q`|컨테이너 ID만 표시|
    |`--size` , `-s`|파일 크기 표시|
    
    |명령결과 항목|설명|
    |---|---|
    |CONTAINER ID|컨테이너 ID|
    |IMAGE|이미지|
    |COMMAND|컨테이너 안에서 실행되고 있는 명령|
    |CREATED|컨테이너 작성 후 경과 시간|
    |STATUS|컨테이너 상태(restarting \| running \| paused \| exited)|
    |PORTS|할당된 포트|
    |NAMES|컨테이너 이름|
    
    |출력형식 플레이스 홀더|설명|
    |---|---|
    |.ID|컨테이너 ID|
    |.Image|이미지 ID|
    |.Command|실행 명령|
    |.CreatedAt|컨테이너가 작성된 시간|
    |.RunningFor|컨테이너의 가동 시간|
    |.Ports|공개 포트|
    |.Status|컨테이너 상태|
    |.Size|컨테이너 디스크 크기|
    |.Names|컨테이너명|
    |.Mounts|볼륨마운트|
    |.Networks|네트워크명|

# 4.3 Docker 컨테이너 네트워크

Docker 컨테이너끼리 통신을 할 때는 Docker 네트워크를 통해 수행한다.

## **네트워크 목록 표시(docker network ls)**

```docker
docker network ls [옵션]

# 브리지 네트워크의 네트워크 ID만을 목록으로 표시
docker network ls -q --filter driver=bridge
```

|옵션|설명|
|---|---|
|`-f` , `--filter=[]`|출력 필터링|
|`--no-trunc`|상세 정보 출력|
|`-q` , `--quiet`|네트워크 ID만 표시|

|필터링 key|설명|
|---|---|
|`driver`|드라이버 지정|
|`id`|네트워크 ID|
|`label`|네트워크에 설정된 라벨  <br>label=<key> 또는 label=<key>=<value>로 지정한다.|
|`name`|네트워크명|
|`scope`|네트워크 스코프(swarm/ global/ local)|
|`type`|네트워크 타입(사용자 정의 네트워크 custom/정의 완료 네트워크 builtin)|

## **네트워크 작성(docker network create)**

```docker
docker network create [옵션] <네트워크명>

# 'web-network' 브리지 네트워크 작성
docker network create --driver=bridge web-network
```

|옵션|설명|
|---|---|
|`--driver` , `-d`|네트워크 브리지 또는 오버레이 (기본값 bridge)|
|`--ip-range`|컨테이너에 할당하는 IP 주소 범위 지정|
|`--subnet`|서브넷을 CIDR 형식으로 지정|
|`--ipv6`|IPv6 네트워크 유효화 여부 (true/ false)|
|`label`|네트워크에 설정하는 라벨|

  

## **네트워크 연결(docker network connect/docker network disconnect)**

```docker
docker network connect [옵션] <네트워크> <컨테이너>
docker network disconnect [옵션] <네트워크> <컨테이너>

# 'webfront' 컨테이너를 'web-network' 네트워크에 연결
docker network connect web-network webfront

# 컨테이너 시작 시 네트워크 지정
docker container run -itd --name=webap --net=web-network nginx

# 네트워크 연결 해제
docker network disconnect web-network webfront
```

|옵션|설명|
|---|---|
|`--ip`|IPv4 주소|
|`--ip6`|IPv6 주소|
|`--alias`|별칭|
|`--link`|다른 컨테이너에 대한 링크|

  

## **네트워크 상세 정보 확인(docker network inspect)**

```docker
docker network inspect [옵션] <네트워크>

# 'web-network' 네트워크 상세 정보 확인
docker network inspect web-network
```

  

## **네트워크 삭제(docker network rm)**

```docker
docker network rm [옵션] <네트워크>

# 'web-network' 네트워크 삭제
docker network rm web-network
```
