## Docker 이미지 다운로드하기

Docker 컨테이너를 실행하기 위해서는 'Docker 이미지'가 필요하다. Docker의 공식 ~~==리포지토리==인~~ ==레지스트리==인 Docker Hub에서 Nginx 공식 이미지를 다운로드 받을 수 있다.

  

### IT 용어 설명

- **도커 리포지토리(Docker Repository):** 특정 이름을 가진 도커 이미지들의 모음. 단일 이미지의 여러 버전(태그: tag)로 구성됨
    
    - 예시
        
        - `nginx` 리포지토리에는 `nginx:1.23`, `nginx:latest`, `nginx:1.22-alpine` 등 여러 버전의 이미지가 있음
        
    
    - 역할: 버전별로 이미지를 관리하여 특정 태그를 통해 필요한 버전의 이미지를 쉽게 가져올 수 있게 함
    

- **도커 레지스트리(Docker Registry):** 도커 이미지를 저장하고 배포하는 서버나 서비스. 여러 개의 도커 리포지토리를 포함할 수 있음
    
    - 예시
        
        - 공개 레지스트리: Docker Hub, Google Container Registry(GCR), Amazon Elastic Container Registry(ECR) 등
        
        - 사설 레지스트리: 조직 내부에서 도커 이미지를 관리하기 위해 사용되는 자체 호스팅된 레지스트리
        
    
    - 역할: 도커 클라이언트(로컬 환경)와 이미지를 저장하는 서버 간에 이미지를 업로드(push)하거나 다운로드(pull)할 수 있도록 지원함
    

  

### Nginx 이미지 다운로드 (`docker pull nginx`)

```bash
docker pull nginx
```

  

### 다운로드 된 이미지 목록 확인 (`docker image ls`)

```bash
docker image ls
```

- 출력 결과
    
    - `REPOSITORY`: 이미지 이름
    
    - `TAG`: 이미지 태그명
    
    - `IMAGE ID`: 이미지 ID
    
    - `CREATED`: 작성일
    
    - `SIZE`: 이미지 크기

## Nginx를 작동시켜 보자

Docker 컨테이너의 바탕이 되는 Docker 이미지를 사용하여 Nginx 서버를 가동시켜 보자.

### 예) 'nginx' 이미지를 사용하여 Docker 컨테이너 'webserver' 실행

```bash
docker container run --name webserver -d -p 80:80 nginx
```

- `-p 80:80`: 브라우저에서 HTTP(80번 포트)에 대한 액세스 허가

- 출력 결과: 컨테이너 ID

## Nginx 작동 확인

### 1) 명령으로 확인

### 실행중인 컨테이너 목록 확인 (`docker container ps`)

> `docker container ls` 명령도 동일한 기능을 한다.

```bash
docker container ps
```

![[89p.실행결과_1.png]]

- 출력 결과
    
    - `CONTAINER ID`: 컨테이너 ID
    
    - `IMAGE`: 컨테이너의 바탕이 된 이미지
    
    - `COMMAND`: 컨테이너 안에서 실행되고 있는 명령
    
    - `CREATED`: 컨테이너 작성 후 경과 시간
    
    - `STATUS`: 컨테이너의 상태 (restarting | running | paused | exited)
    
    - `PORTS`: 할당된 포트
    
    - `NAMES`: 컨테이너 이름
    

  

### 컨테이너의 상세 내용 확인 (`docker container stats [컨테이너 이름]`)

```bash
docker container stats [컨테이너 이름]
```

![[89p.실행결과_2(docker_container_stats_webserver).png]]

- 출력 결과
    
    - `CONTAINER ID`: 컨테이너 ID
    
    - `NAME`: 컨테이너 이름
    
    - `CPU %`: CPU 사용률
    
    - `MEM USAGE / LIMIT`: 메모리 사용량 / 컨테이너에서 사용할 수 있는 메모리 제한
    
    - `MEM %`: 메모리 사용률
    
    - `NET I/O`: 네트워크 I/O
    
    - `BLOCK I/O`: 블록 I/O
    
    - `PIDS`: 컨테이너에서 실행중인 프로세스 개수 (Windows 컨테이너 제외)
    

- `Ctrl`+`C`를 눌러 명령 종료

  

### 2) 브라우저에서 확인

Nginx 서버가 정상적으로 작동하는 경우, 브라우저 url로 [http://localhost:80](http://localhost/) 입력후 액세스해보면 Nginx 서버의 탑 화면이 표시된다.

## Nginx의 기동 및 정지

### 컨테이너 정지 (`docker stop <컨테이너 이름>`)

컨테이너가 정지하므로 웹 브라우저에서 액세스하면 오류가 발생한다.

```bash
docker stop webserver
```

![[90p.실행결과_1.png]]

![[90p.실행결과_2.png]]

  

### 컨테이너 재기동 (`docker start <컨테이너 이름>`)

> `docker container start <컨테이너 이름>` 명령도 동일한 기능을 한다.

```bash
docker start webserver
```
