- 이미지 내부에는 인터넷 상에 공개하고 싶지 않은 정보가 포함되는 경우도 있으며, 이러한 경우를 위해 Docker 이미지를 일원 관리하기 위한 레지스트리를 로컬 환경에 구축, 관리하는 방법에 대해 설명

  

### 로컬 환경에 Docker 레지스트리 구축하기

- Docker 레지스트리를 프라이빗 네트워크 안에서 구축하려면 Docker Store에 공개되어 있는 공식 이미지인 ‘registry’를 사용한다.

- ‘registry’ 는 Version0 ( Python ) / Version2 ( GO ) 두 계열의 버전이 있다.

- 두 버전은 호환성이 없으며, 특별한 요구사항이 없는 경우, Version2 계열을 이용하는 것이 좋다.

- Docker registry는 프라이빗 네트워크 안에서만 이미지를 공개할 수 있다.

1. 해당 registry를 docker image pull 명령을 사용해 로컬 환경에 다운로드

1. 다운로드 된 이미지 확인

1. 다운로드한 registry 이미지를 바탕으로 레지스트리용 컨테이너 실행 ( docker run - )

1. 컨테이너가 실행되면 프라이빗 레지스트리 구축

  

### Docker 이미지 업로드

1. Dockerfile 생성

1. 빌드 ( docker build - )
    
    - 프라이빗 네트워크 안의 Docker 레지스트리에 업로드 하려면 규칙을 사용해 태그를 붙여야 한다.
        
        → docker image tag [로컬의 이미지명] [업로드할 레지스트리의 주소:포트 번호]/이미지명
        
    

1. 생성된 이미지를 프라이빗 레지스트리에 업로드 ( docker image push - )

  

### Docker 이미지의 다운로드와 작동 확인

1. docker image pull [레지스트리 주소]/[이미지명] 을 사용해 이미지를 프라이빗 레지스트리에서 로컬 환경으로 다운로드

1. 해당 이미지를 바탕으로 컨테이너 실행 ( docker run - ) 으로 작동 확인

- 주의할 점은, 프라이빗 레지스트리에서는 Docker 이미지를 영구 데이터로 관리할 필요가 있다는 점이다. Docker 에서는 호스트 머신에 볼륨을 공유함으로써 영구 데이터를 일원 관리할 수 있다. 하지만 이 데이터를 관리하려면 신뢰성이 높고 용량이 충분한 스토리지를 마련해야 하며, 만일의 장애에 대비해 다중화 구성을 검토할 필요가 있다. 또한 정기적 백업이나 서버 감시 등의 운용이 필요하다.
