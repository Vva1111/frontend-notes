## FileReader 객체

- 웹 브라우저에서 File / Blob(주로 이미지, 비디오, 사운드 등과 같은 멀티미어 객체)을 비동기적으로 읽을 수 있게 하는 객체이며, 이 객체를 사용해 사용자가 선택한 파일 혹은 Blob의 내용을 읽고 처리할 수 있다.

- 해당 기능을 사용해 파일을 비동기적으로 읽을 수 있으므로 Ajax 요청 없이도 로컬 파일 시스템에서 파일을 가져와서 웹 애플리케이션 내에서 바로 처리할 수 있다. 이를 활용해 사용자가 선택한 파일을 전송하기 전에 미리 확인하거나, 읽어볼 수 있다.

### FileReader 속성

- FileReader
    
    - .error : 파일 읽는 도중 발생한 에러를 나타낸다.
    
    - .readyState : FileReader의 상태를 나타내며,
        
        0 : 아직 데이터가 로드되지 않음
        
        1 : 데이터 로딩 중
        
        2 : 모든 읽기 요청이 완료
        
        를 나타낸다.
        
    
    - .result : 파일의 컨텐츠를 나타낸다. 이 속성은 읽기 작업이 완료되고 읽기 작업의 초기화에 사용한 방식으로 결정된 데이터의 포맷이 정해진 후에 유효하다.
    

  

### FileReader 이벤트 핸들러

- FileReader
    
    - .onabort() : 읽기 작업이 중단 될 때마다 발생
    
    - .onerror : 읽기 작업에 에러가 생길 때마다 발생
    
    - .onload : 읽기 작업이 성공적으로 완료 되었을 때마다 발생
    
    - .loadstart : 읽기 동작이 실행될 때마다 발생
    
    - .loadend : 읽기 동작이 끝났을 때마다 발생( 읽기의 성공이나 실패 여부는 상관하지 않고 발생 )
    
    - .progress : Blob 컨텐츠를 읽는 동안 호출
    

  

### FileReader 메서드

- FileReader
    
    - .abort() : 읽기 요청을 중단시키며, return 할 경우 readyState값은 DONE을 반환한다.
    
    - .readAsArrayBuffer() : 특정 Blob 파일을 ArrayBuffer로 읽는다. ArrayBuffer는 바이너리 데이터를 처리하는데 사용
    
    - .readAsDataURL() : 지정된 Blob의 내용을 URL로 나타낸다. Data URL은 파일의 내용을 인코딩한 문자열로 표현한다.
