개발 환경과 제품 환경에서 필요한 모듈이 다르므로 멀티스테이지 빌드를 활용하는 것이 좋다.

# Dockerfile 만들기

```docker
# -- 1. 개발 환경용 이미지 -------------------------------------
# 1) Base Image
# 개발용 언어 Go를 베이스 이미지로 지정
FROM golang:1.8.4-jessie AS builder

# 2) Install dependencies
# 작업 디렉토리 이동. 이후 명령이 이 디렉토리를 기준으로 실행됨
WORKDIR /go/src/github.com/assashiho/greet
# 해당 url의 라이브러리 다운로드
RUN go get -d -v github.com/urfave/cli

# 3) Build modules
# 로컬 환경에 있는 main.go 파일을 개발 환경용 작업 디렉토리로 복사
COPY main.go .
# Linux용 바이너리를 빌드하여 'greet'라는 이름의 실행 가능한 바이너리 파일 생성
RUN GOOS=linux go build -a -o greet .

# -- 2. 제품 환경용 이미지 -------------------------------------
# 1) Base Image
FROM busybox
# 작업 디렉토리 이동. 이후 명령이 이 디렉토리를 기준으로 실행됨
WORKDIR /opt/greet/bin

# 2) Deploy modules
# 개발 환경용 이미지로 빌드한 'greet'을 제품 환경용 작업 디렉토리로 복사
COPY --from=builder /go/src/github.com/assashiho/greet/ .
# 'greet' 파일 실행
ENTRYPOINT ["./greet"]
```

- `-from=builder`: 이름이 `builder`인 이미지로부터

> [!important]
> 
> **busybox**
> 
> 기본적인 Linux 명령을 하나의 파일로 모아놓은 것
> 
> - 최소한으로 필요한 Linux 쉘 환경을 제공할 때 사용

  

---

# Docker 이미지의 빌드

```bash
docker build -t greet .
```

> [!important]
> 
> ==**책 설명과 실행 결과가 다른 부분 (Ubuntu 22.04)**==
> 
> 책 내용과 달리 중간 이미지는 `docker image ls` 명령을 통해 확인 불가

# Docker 컨테이너의 시작

제품 환경용으로 만든 저용량 이미지인 'greet'만으로 동작한다.

```bash
$ docker container run -it --rm greet asa
Hello asa

$ docker container run -it --rm greet --lang=es asa
Hola asa
```

> [!important]
> 
> ==**책 설명과 실행 결과가 다른 부분 (Ubuntu 22.04)**==
> 
> 위 Dockerfile 내용으로 컨테이너 시작이 제대로 되지 않고 `docker: Error response from daemon: failed to create task for container: failed to create shim task: OCI runtime create failed: runc create failed: unable to start container process: exec: "asa": executable file not found in $PATH: unknown.` 에러가 발생한다.
> 
>   
> 
> > 문제 해결 시도
> 
> 1. Dockerfile `COPY --from=builder /go/src/github.com/assashiho/greet/ .` → `COPY --from=builder /go/src/github.com/assashiho/greet/greet .` 변경
>     
>     - 일부 내용이 누락되었나 싶어 변경해 봄
>     
> 
> 1. Dockerfile `RUN GOOS=linux go build -a -o greet` → `RUN CGO_ENABLED=0 GOOS=linux go build -a -o greet .` 변경
>     
>     - busybox에서 동작하도록 정적으로 빌드하는 방법
>
