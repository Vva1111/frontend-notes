Docker 이미지를 만들려면 미들웨어 설치, 사용자 계정/디렉토리 작성 등의 명령을 실행해야 한다. 혹은, 이미지로부터 컨테이너를 생성했을 때 서버 프로세스 등을 [[5.4 명령 및 데몬 실행]]으로 작동시킬 필요도 있다.

  

> [!important]
> 
> - 데몬이란?
>     
>     백그라운드에서 실행되는 프로그램.
>     
>     사용자가 실행하지 않아도 자동으로 실행되며, 시스템을 계속해서 모니터링 하거나 특정 작업을 수행한다.
>     
>     - 데몬의 예
>         
>         - 웹서버 - nginx, apache
>         
>         - DB - mySqld
>         
>         - Docker **-** dockerd
>         
>     
>       
>     
>     nginx는 기본적으로 데몬으로 실행되지만, Docker 컨테이너는 포그라운드 프로세스가 1개 이상 필요하다. 따라서 `daemon off` 옵션을 이용해 포그라운드 모드로 실행해줘야 한다.
>     
>     Apache의 경우 `-D FOREGROUND` 옵션을 사용한다.
>

## **🌊 RUN 명령 (명령 실행)**

> [!important]
> 
> `RUN <실행하고 싶은 명령>`

- FROM 명령에서 지정한 베이스 이미지에 대해 명령을 실행할 때 사용  
    ➡ Docker 이미지를 생성할 때 실행

- Dockerfile 작성 시 가장 많이 사용하는 명령

- 하나의 Dockerfile에 여러 개의 RUN 명령을 기술 가능
    
    - 기술된 내용에 따라 순차적으로 실행
    
    - 명령 한 줄마다 이미지(레이어)를 생성
    

  

### ✔ **RUN 명령의 두 가지 기술 방법**

**1️⃣ Shell 형식**

- 일반적인 리눅스 명령어 실행 방식
    
    - 명령 인수에 환경 변수 사용 가능
    

- `sh -c` (리눅스 기본 쉘)를 통해 실행됨
    
    - Docker 컨테이너 안에서 `/bin/sh -c` 를 사용해 명령 실행할 때와 동일한 작동
    

- 기본 쉘을 변경하고 싶을 때는 [[5.5 환경 및 네트워크 설정]]을 사용한다 (5.5장)

```docker
# Nginx의 설치
RUN apt-get install -y nginx
```

  

**2️⃣ Exec 형식**

> [!important]
> 
> `["<실행할 프로그램>", "<인수 1>", ...]`

- 쉘을 거치지 않고 바로 실행됨
    
    - 환경 변수 지정 불가
    

- JSON 배열 형태로 작성

- 다른 쉘을 이용하고 싶을 땐 RUN 명령에서 쉘 경로를 지정

- 문자열을 인수로 지정할 때는 `'(홑따옴표)` 사용

```docker
# Nginx의 설치
RUN ["/bin/bash", "-c", "apt-get install -y nginx"]

# echo 사용
RUN ["echo", " 안녕하세요 Exec 형식입니다"]
```

- `bash -c`
    
    위 예제처럼 `bash -c`를 사용하는 경우, 쉘 기능(`&&`, `|`, `$VAR` 등)을 포함한 명령어도 실행할 수 있다. `bash -c "<여기 있는 명령어를 그대로 실행>"` 시키는 것이기 때문.
    

  

> [!important]
> 
> **RUN 명령**은 Docker 이미지가 빌드되는 과정에서 실행되며, 실행 중 쉘 기능을 활용하는 경우가 많다. 패키지를 설치하거나, 파일을 복사하고, 여러 명령어를 한 줄로 결합하는 경우에도 쉘 기능을 사용한다.
> 
> **Exec 형식**은 쉘을 통하지 않고 명령이 실행되므로 성능이나 보안 측면에서 유리하지만 쉘 기능을 사용할 수 없다.
> 
> 따라서, 일반적으로 ==RUN 명령에서는 **Shell 형식**이 더 많이 사용==된다. **Exec** 형식은 쉘 기능을 사용할 필요가 없고, 단순히 실행할 프로그램을 지정할 때 유용하다.

  

> [!important]
> 
> **이미지의 레이어에 대해**
> 
> ---
> 
> Dockerfile을 빌드하면 기술된 명령어마다 내부 이미지가 하나씩 작성된다.
> 
> ```docker
> # 4개의 레이어가 생성됨
> RUN yum -y install httpd
> RUN yum -y install php
> RUN yum -y install php-mbstring
> RUN yum -y install php-pear
> 
> # 하나의 레이어만 생성됨
> RUN yum -y install httpd php php-mbstring php-pear
> ```
> 
> 이와 같이 한 줄에 쓸 수 있는 RUN 명령은 한 줄에 쓰는 것이 좋다. 또, `\`로 줄바꿈을 넣어 가독성을 높일 수 있다.
> 
> ```docker
> RUN yum -y install httpd\
> 									 php\
> 									 php-mbstring\
> 									 php-pear
> ```

## **🌊 ENTRYPOINT 명령 (데몬 실행)**

> [!important]
> 
> `ENTRYPOINT <실행하고 싶은 명령>`

- docker container run 명령을 실행했을 때 실행됨

  

### ✔ **ENTRYPOINT 명령의 두 가지 기술 방법**

**1️⃣ Exec 형식** _==※ RUN 명령의 구문과 동일==_


  

**2️⃣ Shell 형식** _==※ RUN 명령의 구문과 동일==_

- 일반적인 리눅스 명령어 실행 방식
    
    - 명령 인수에 환경 변수 사용 가능

```docker
# 서버 실행
ENTRYPOINT nginx -g "daemon off;"
```

  

### ✔ **CMD와 ENTRYPOINT 차이점**

|CMD|ENTRYPOINT|
|---|---|
|실행할 기본 명령을 설정|필수 실행 명령을 설정|
|docker run 시 다른 명령으로 대체 가능|docker run 시 다른 명령으로 대체 불가==※ CMD 명령과 조합하면 인수 지정 가능==|

  

### ✔ **CMD와 조합**

ENTRYPOINT 명령 실행 시 명령 인수를 지정하고 싶을 때 CMD와 조합해서 사용함

이때 CMD 명령에서 지정한 옵션을 사용하여 실행 시의 인수를 임의로 `docker container run` 명령 실행 시로 지정할 수 있음.

- 즉, **ENTRYPOINT는 실행할 명령을 정의하고**, **CMD는 그 명령에 기본 인수를 제공**하는 역할을 함

  

```docker
# Docker 이미지 취득
FROM ubuntu:16.04

# top 실행
ENTRYPOINT ["top"]
CMD ["-d", "10"]
```

```bash
# CMD 명령에서 지정한 10초 간격으로 갱신
$ docker container run -it sample

# 2초 간격으로 갱신
$ docker container run -it sample -d 2
```

## **🌊 STOPSIGNAL 명령 (시스템 콜 시그널 설정)**

> [!important]
> 
> `STOPSIGNAL <시그널>`

- 컨테이너 종료 시 송신하는 시그널을 설정
    
    - 기본적으로 `SIGTERM` 시그널을 보내 프로세스 종료를 요청함
    
    - 특정 시그널을 보내야 하는 경우 STOPSIGNAL 명령을 사용하여 변경하는 것
    

- 시그널 번호 또는 시그널명 지정 가능

  

> [!important]
> 
> **주요 시그널 목록**
> 
> |시그널명|시그널 번호|설명|
> |---|---|---|
> |`SIGTERM`  <br>(default)|15|프로세스를 정상적으로 종료 요청|
> |`SIGKILL`|9|프로세스 강제 종료.  <br>프로세스 정리 작업 없이 즉시 종료.|
> |`SIGINT`|2|인터럽트 시그널.  <br>`Ctrl + C`와 비슷한 방식으로 애플리케이션에 중단 요청.  <br>  <br>`SIGKILL` 보다 안전하게 종료 가능|

## **🌊 HEALTHCHECK 명령 (컨테이너 헬스 체크 명령)**

> [!important]
> 
> `HEALTHCHECK [옵션] CMD <실행할 명령>`

- 컨테이너 안의 프로세스의 정상 작동 여부 확인

- 정상 작동 시 상태를 `healthy`로, 아닐 시 `unhealthy`로 표시

- `docker container inspect` 명령으로 결과 확인

  

```docker
HEALTHCHECK --interval=5m --timeout=3s CMD curl -f http://localhost/ || exit 1
```
