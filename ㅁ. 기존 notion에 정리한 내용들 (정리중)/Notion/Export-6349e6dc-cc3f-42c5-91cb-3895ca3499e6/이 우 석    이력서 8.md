## ==자기 소개==

Vue 3, TypeScript, Quasar, ECharts 등을 활용한 기업 내 SaaS 자산 통합 관리 기능과 가상 머신 리소스 사용량, 비용, 사용 추이 데이터를 시각화하는 대시보드·미터링·리포트 화면을 개발하며, 운영 지표를 직관적으로 제공하고 사용자의 데이터 기반 의사결정을 지원하는 화면을 구현해 왔습니다.

운영자용 백오피스 화면에서 검색, 필터, 정렬, 페이징, 가상 스크롤, 상세 조회, 데이터 다운로드 기능 등을 구현했으며, 권한 관리 및 인증 영역에서 OIDC 기반 다중 SPA 간 인증 정보 연동과 API Gateway 기반 프론트엔드 인증/인가 처리 경험을 보유하고 있습니다.

  

제품 설계 및 구현 과정에서는 페이지 로드 시간을 약 2~3초 단축해 **사용자 편의성을 개선**하고**,** GitLab CI/CD, Docker, K8S를 활용한 배포 파이프라인 구성으로 코드 통합·빌드·배포 시간을 10여 분 에서 3분대로 약 70% 가량 단축하여 **개발 생산성을 향상시킨** 경험이 있습니다.

  

  

[ **AI 페어 프로그래밍을 통한 렌더링 병목 현상 해결 및 리팩토링** ]

실무 개발 과정에서 AI를 단순한 코드 생성기가 아닌, 구조 설계와 문제점 해결을 함께 논의하는 '페어 프로그래머'로 활용하며 개발 생산성을 높이고 있습니다.

대규모의 가상 머신 자원 정보를 테이블 컴포넌트에 바인딩하는 과정에서 심각한 렌더링 지연 현상을 겪었던 경험이 있습니다. 원인을 찾기 위해 구현되어 있는 Vue 3의 상태 관리 로직과 화면 렌더링 방식 구성을 명확한 컨텍스트로 AI에게 제공하고, 기술적 조언을 구했습니다.

AI와의 문답을 통해 분석한 결과, Vue가 대용량 배열 내부 객체들의 깊은 반응성까지 모두 추적하면서 발생하는 재귀적 연산 오버헤드가 병목의 핵심 원인임을 확인했습니다. 이를 해결하기 위해 기존 `ref`를 최상위 수준의 반응성만 갱신하는 `shallowRef`로 대체하는 방향으로 상태 관리 함수를 리팩토링했습니다.

결과적으로 불필요한 반응성 추적 연산을 제거하여 테이블 렌더링 지연과 무한 루프 에러를 해결할 수 있었습니다. 이를 통해 직접 해결 방법을 찾아보며 교차 검증하는데 소요됐을 디버깅 시간을 대폭 단축할 수 있었습니다.

이러한 실무형 프롬프트 엔지니어링 역량을 바탕으로, 개발을 직접 주도하되, 동료 개념의 AI 에이전트 사용으로 업무에 유연하게 통합해 개발 속도와 코드 품질을 동시에 끌어 올릴 수 있는 역량을 보유하고 있습니다.

  

[ **로딩 속도 개선을 통한 사용자 경험 강화** ]

페이지 로딩 속도 개선으로 사용자 경험을 향상시켰습니다.

페이지 최초 접근 시 최대 5초 수준의 과도한 로딩 시간이 발생하는 문제가 있었습니다. 네트워크 탭을 통해 로드 시간을 분석한 결과, 프로젝트에서 외부 리소스로 import하던 폰트와 아이콘 이미지가 실제 사용 여부와 관계없이 일괄 로딩되고 있어 초기 페이지 진입 시 불필요한 네트워크 요청과 리소스 로딩 비용이 발생하고 있음을 확인했습니다.

이를 해결하기 위해 전체 폰트와 이미지를 일괄 import하는 방식을 중단하고, 프로젝트에서 실제 사용하는 폰트와 이미지만 선택적으로 로딩할 수 있도록 리소스 관리 구조를 라이브러리화했습니다.

그 결과 불필요한 네트워크 요청과 초기 로딩 비용을 줄였으며, **초기 페이지 로딩 시간을 약 2,000~3,000ms 단축**하는 개선 효과를 얻었습니다.

  

[ **배포 파이프라인 개선 및 GitLab 을 활용한 자동화 구성으로 개발 업무 편의성 증가** ]

프론트엔드, 백엔드, 배포 코드가 혼재된 레거시 프로젝트의 배포 구조를 개선했습니다.

프론트엔드, 백엔드, 배포 코드가 모두 포함된 레거시 프로젝트의 통합/배포 과정에서 약 13분이 소요되어 업무 병목과 분리되지 않은 구조로 인해 병합 충돌도 빈번하게 발생해 개발 생산성에 영향을 주고 있었습니다.

문제를 해결하기 위해 프론트엔드 코드만으로 관심사를 분리하고 패키지 캐싱, 빌드 산출물의 크기 제한 및 파일 분류, CI/CD 분기 처리, 별도 이미지 빌드 구조를 적용해 배포 파이프라인을 대폭 개선 했습니다.

그 결과로 **10여 분이 걸리던 통합/배포 전 과정을 3분 대로 줄여 70%의 시간 감소 효과**를 얻고 업무 효율을 상승시킬 수 있었습니다.
