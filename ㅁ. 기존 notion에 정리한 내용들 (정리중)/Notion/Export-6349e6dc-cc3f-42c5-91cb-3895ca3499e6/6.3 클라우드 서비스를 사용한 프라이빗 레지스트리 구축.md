- Docker 이미지는 인프라 구성 요소에서 애플리케이션의 개발 환경 및 실행 모듈도 포함하기 때문에 용량이 큰 것들이 있을 수 있다.

- 이러한 이미지를 모두 개발용 클라이언트 PC 혹은 온프레미스 환경에서 관리하려면 비용과 시간이 많이 든다.

- 퍼블릭 클라우드에는 Docker 이미지를 프라이빗으로 관리할 수 있는 매니지드 서비스가 제공되므로, 이것을 이용하는 것이 좋다.

- 해당 책에서는 Google Cloud Platform 에서 제공하는 프라이빗 이미지 관리 서비스인 Google Container Registry 를 사용해 Dockerfile로부터 생성한 독자적인 Docker 이미지를 관리하는 방법을 설명하고 있다.

  

### Google Container Registry 준비하기

- Google Container Registry 를 이용하려면 API 사용을 허가해야 한다.

  

### Docker 이미지 업로드

- Google Container Registry 에 이미지를 업로드 한다.

- Google Container Registry 에 이미지를 업로드 하기 위해서는 아래와 같은 태그 규칙이 필요하다.
    
    ⇒ docker tag [로컬 이미지명] [Google Container Registry의 호스트명]/[프로젝트 ID]/[이미지명]
    

- 호스트명은 저장 위치에 따라 미국, 유럽, 아시아 중 하나를 선택한다.

- 이미지를 레지스트리에 push 할 때는 gcloud 라는 명령어가 필요하다.
    
    ```tsx
    gcloud docker -- push asia.gcr.io/docker-book/-
    ```
    
    - 클라이언트 PC에서 gcloud 명령을 실행하려면 미리 Google SDK를 성치하고 인증을 받을 필요가 있다.
    

  

### Docker 이미지의 다운로드와 작동 확인

- Google Container Registry 상에 있는 이미지를 로컬 환경으로 다운로드 하려면 push와 마찬가지로 gcloud 명령으로 실행해야 한다.

- 다운로드 후, 이미지 목록 확인과 컨테이너 실행으로 정상 작동 여부를 확인한다.
