- Dockerfile을 Github 등에서 관리하고 Docker Hub와 연결하면 Dockerfile 로부터 자동으로 Docker 이미지를 생성하고 공개할 수 있다.

  

### Automatic Build의 흐름

- Docker Hub에는 버전 관리 툴인 GitHub 등과 연결해 Dockerfile 로부터 Docker 이미지를 자동으로 생성하는 ‘Automatic Build’ 기능이 있다.
    
    - 이 기능은 GitHub 또는 BitBucket에서 관리되는 Dockerfile을 바탕으로 Docker 이미지를 자동으로 빌드하는 기능을 말한다.

### GitHub에 공개하기

- 작성한 Dockerfile을 GitHub에서 공개한다.

- 애플리케이션의 소스코드 관리와 똑같은 절차로 리포지토리를 만들어 Dockerfile을 공개한다.

- 이 때, 반드시 Dockerfile 이라는 파일로 리포지토리에 공개해야 한다.

  

### Docker Hub의 링크 설정 (GitHub)

1. Docker Hub에 로그인

1. Docker Hub의 메인 화면에서 계정 버튼 클릭

1. [Settings] → [Linked Accounts & Service] 클릭

1. [Link GitHub] 클릭

1. GitHub의 계정과의 연결 방법 설정 (Public and Private - Recommended)
    
    - 해당 설정으로 GitHub의 프라이빗 리포지토리에서도 자동으로 빌드를 할 수 있다.
    
    - Limited Access : 퍼블릭 리포지토리 외에서는 수동으로 빌드해야 한다.
    

1. [Select] 클릭

1. GitHub 계정과 연결

  

### Dockerfile의 빌드

- GitHub 상의 Dockerfile을 바탕으로 Docker Hub에서 이미지를 생성한다.

  

1. Docker Hub 로그인

1. [Create] 클릭

1. [Create Automated Build] 클릭

1. Dockerfile이 공개되어 있는 GitHub 계정 선택

1. 4를 선택하면 공개되어 있는 소스코드의 리포지토리가 표시되므로, 여기서 리포지토리를 선택한다.

1. 만들 Docker 이미지의 정보를 등록
    
    - Repository Namespace & Name ( 필수 )
        
        - Docker 이미지의 이름 공간과 Docker Hub에서 이미지를 공개할 때의 이름
        
    
    - Visibility ( 필수 )
        
        - Docker 이미지 공개 여부 설정
        
    
    - Short Descrition ( 필수 )
        
        - Docker 이미지에 대한 간단한 설명
        
    

1. [Create] 클릭

1. 리포지토리 화면으로 이동해 [Build Settings] 클릭
    
    - Docker 이미지의 태그를 설정
    

1. [Trigger] 버튼을 클릭하면 빌드 시작

⇒ Docker Hub 의 [Build Settings] 에서 [When active, builds will happen automatically on pushes.] 에 선택 표시를 하면 Dockerfile을 저장하고 있는 GitHub의 내용이 갱신된 경우 Docker 이미지의 빌드가 자동으로 일어난다.
