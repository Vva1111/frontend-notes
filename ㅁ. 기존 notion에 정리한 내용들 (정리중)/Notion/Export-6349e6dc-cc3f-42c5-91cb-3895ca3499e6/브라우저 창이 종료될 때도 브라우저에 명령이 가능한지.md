> HTML 문서의 생명주기엔 다음과 같은 3가지 주요 이벤트가 관여합니다.  
> [https://ko.javascript.info/onload-ondomcontentloaded](https://ko.javascript.info/onload-ondomcontentloaded)  

> [!info] Fetch API  
> We want to make this open-source project available for people all around the world.  
> [https://javascript.info/fetch-api](https://javascript.info/fetch-api)  

- HTML 문서의 생명 주기에는 다음과 같은 3가지의 주요 이벤트가 관여한다.
    
    - DOMContentLoaded : 브라우저가 HTML을 전부 읽고 DOM 트리를 완성하는 즉시 발생하는 이벤트 ( 이미지 파일<img> 이나 stylesheet 등의 기타 자원은 기다리지 않는다. )
    
    - load : HTML로 DOM 트리를 만드는 것이 완성되었을 뿐만 아니라 이미지, stylesheet 와 같은 외부 자원도 모두 불러오는 것이 끝났을 때 발생하는 이벤트
    
    - **beforeunload / unload** : 사용자가 페이지를 떠날 때 발생하는 이벤트
    
      
    

## unload

- window 객체의 unload 이벤트는 사용자가 페이지를 떠날 때, 즉 문서를 완전히 닫을 때 실행된다. unload 이벤트에선 팝업 창을 닫는 것과 같은 딜레이가 없는 작업을 수행할 수 있다.

- unload 이벤트는 사용자가 페이지를 떠날 때 자연스럽게 발생한다.

- 하지만 페이지가 종료될 때 분석 정보를 보내는 것은 예외 사항에 속하는데,
    
    사용자가 페이지를 떠날 때 분석 정보를 서버로 전송하는 이벤트는 navigator.sendBeacon(url, data) 를 사용할 수 있다.
    

- sendBeacon은 데이터를 백그라운드에서 전송한다. 다른 페이지로 전환 시 분석 정보는 제대로 서버에 전송되지만, 딜레이는 없으며, 아래와 같이 사용한다.

```jsx
let analyticsData = { /* 분석 정보가 담긴 객체 */ };

window.addEventListener("unload", function() {
  navigator.sendBeacon("/analytics", JSON.stringify(analyticsData));
};
```

- 요청은 POST로 전송되며, 문자열 뿐만 아니라 fetch에서 지원하는 기타 포맷들도 보낼 수 있다.
    
    ( 전송 데이터는 64kb를 넘을 수 없다. )
    

- sendBeacon 요청이 종료 된 시점엔 브라우저가 다른 페이지로 전환을 마친 상태일 확률이 높기 때문에 응답을 받을 수 없는 상황이다.

  

- fetch 메서드는 ‘페이지를 떠난 후에도’ 요청이 가능하도록 해주는 플래그 keepalive를 지원한다.

> [!important] Fetch - keepalive
> 
> - Fetch를 통한 요청이 요청을 시작한 웹 페이지보다 오래 지속될 수 있게 하는 옵션
> 
> - 예를 들면, 현재 방문자가 우리 페이지를 사용하는 방법(마우스 클릭, 보고 있는 페이지 등) 에 대한 통계를 수집해 사용자 경험을 분석, 개선하는데 사용할 수 있다.
>     
>     위와 같은 데이터를 ‘방문자가 우리 페이지가 떠날 때’ 시점에 서버에 저장하고 싶을 경우.
>     
> 
> - window.onunload 와 함께 사용할 수 있다.
> 
> ```jsx
> window.addEventListener("unload", function (){
> 	fetch('/analytics', {
> 		method: 'POST',
> 		body: 'statistics',
> 		keepalive: true
> 	}
> });
> ```
> 
> - 이 요청은 브라우저가 페이지를 떠난 후에도 백그라운드에서 요청을 수행하도록 지시한다.
> 
> - 따라서 이 옵션은 요청이 성공한다는 조건이 필수이다.
> 
> - 해당 옵션은 몇 가지 조건이 있는데,
>     
>     - 메가바이트 단위 크기의 데이터는 보낼 수 없다(본문 제한 64KB)
>     
>     - 크기 제한은 모든 keepalive 요청에 함께 적용된다. 즉, keepalive 요청을 병렬로 수행할 수 있지만 모든 keepalive 요청 본문 길이의 합이 64KB를 초과해서는 안된다.
>     
>     - document가 종료되면 서버 응답을 처리할 수 없다. 위 예제에서 Fetch는 keepalive 옵션 덕분에 성공 하지만, 이 이후의 후속 기능은 일어나지 않는다.
>         
>         ( 하지만 통계 데이터를 보내기 위해 정보를 전송하는 경우, 이러한 요청에 대해 대부분 빈 응답을 보내기 때문에 크게 문제가 되진 않는다. )
>         
>     

  

## onbeforeunload

- 사용자가 현재 페이지를 떠나 다른 페이지로 이동하려 하거나 창을 닫으려 할 때 beforeunload 핸들러에서 추가 확인을 요청할 수 있다.

```jsx
window.onbeforeunload = function() {
  return false;
};
```
