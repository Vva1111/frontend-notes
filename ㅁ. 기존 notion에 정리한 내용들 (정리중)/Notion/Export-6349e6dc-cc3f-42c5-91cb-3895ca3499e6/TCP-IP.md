- Internet Protocol Suite

- 인터넷 혹은 이와 유사한 컴퓨터 네트워크 사이에서 정보를 주고받는 데 필요한 일정한 규칙(통신 프로토콜)

- 현재 기본 프로토콜은 전송제어 프로토콜(TCP)과 인터넷 프로토콜(IP)이 있다

  

- 정의
    
    - TCP/IP는 온라인 상의 안전하고 효율적인 데이터 전송의 필수 요건을 정의한다
    
    - 두 대의 기기 간 데이터 전송을 담당
    
    - 인터넷 데이터 전송 방식은 항상 기기 대 기기, 즉, 2대의 기기만 개입된다.
    
    - 4계층 또는 5계층으로 구성되어 있다.
    
    - OSI 7계층 모델을 대체한다
    
    - TCP와 IP 각각 담당하는 작업이 따로 있지만, 결국 같은 결과를 목표로 하기 때문에 하나의 명칭으로 통합해서 주로 사용한다.
    

  

- TCP/IP의 4 계층
    
    - TCP/IP는 통신 규칙의 모음이며, 프로토콜 스위트로 불리기도 한다.
    
    - ‘TCP/IP 4 계층’ 은 이러한 규칙이나 프로토콜이 적용되는 특정한 조건을 의미한다.
    
    - TCP/IP 모델이 다양한 기기와 앱에서 효율적으로 데이터를 주고받을 수 있도록 하는 방식
        
        - 응용 계층
            
            → 앱에 구축되기 때문에 사용자가 상호작용하기 가장 쉬운 계층
            
            → 응용 계층은 사용자( 사람 혹은 소프트 웨어)가 네트워크에 접근할 수 있도록 한다.
            
            → 사용자 인터페이스 제공
            
            → 이메일, 원격 파일 접근 및 전송, 공유, 데이터베이스 관리 등의 서비스 제공
            
            → 메일 프로그램의 경우 인터넷 전자우편 표준 통신 규약인 Simple Mail Transfer
            
            Protocol(SMPT) 이용
            
            → 인터넷 브라우저는 웹 서버와 사용자의 인터넷 브라우저 사이에서 문서를 전송하기
            
            위해 Hyper Text Transfer Protocol(HTTP) 통신 규약 사용
            
            - 위의 두 프로토콜 외에도 흔히 사용되는 응용 계층 프로토콜로는,
                
                - [FTP](https://ko.wikipedia.org/wiki/%ED%8C%8C%EC%9D%BC_%EC%A0%84%EC%86%A1_%ED%94%84%EB%A1%9C%ED%86%A0%EC%BD%9C) : 파일 전송 규약
                
                - [DHCP](https://nordvpn.com/ko/blog/what-is-dhcp/) : 동적 호스트 설정 통신 규약
                
                - [SNMP](https://ko.wikipedia.org/wiki/%EA%B0%84%EC%9D%B4_%EB%A7%9D_%EA%B4%80%EB%A6%AC_%ED%94%84%EB%A1%9C%ED%86%A0%EC%BD%9C) : 간이 망 관리 프로토콜
                
                등이 있다.
                
            
              
            
        
        - 전송 계층 ( TCP / UDP )
            
            → 전송을 담당하는 계층
            
            → 전송 계층 프로토콜은 운영체제 내부 기능으로 구현되기 때문에,
            
            이 서비스를 사용하려면 상위 계층에서 시스템 콜이라는 프로그램 호출 방식을
            
            이용해야 한다.
            
            → 전송 계층에는 TCP 뿐만 아니라, User Datagram Protocol(UDP)도 있다
            
            - 비 연결형 서비스
            
            - UDP는 TCP보다 단순하며, 비교적 보호의 필요성이 적은 데이터에 이용
            
            - TCP보다 신뢰도가 낮고, 오류 검출, 흐름 제어 등의 기능을 제공하지 않아 패킷을 빠르게 전송하는 응용 계층에서 이용된다.
            
              
            
            → TCP는 두 네트워크 사이에 연결을 형성하고, 효율적인 작업을 위해 데이터를 작은
            
            패킷으로 나눠서 전송한다.
            
            → TCP는 신뢰도가 높지만, 속도가 느리며, UDP는 신뢰도가 낮고, 속도가 빠르다
            
            → TCP의 패킷 교환 방식은 가상 회선 방식이고, UDP는 데이터 그램 방식을 따른다.
            
            → TCP는 전송 순서를 보장하지만, UDP의 경우 전송 순서는 바뀔 수 있다.
            
            → TCP에서는 데이터를 ‘세그먼트’라는 블록 단위로 분할해 전송한다.
            
            전송되는 블록의 크기는 네트워크 부하 정도, 윈도우 크기 등의 영향을 받으며,
            
            가변 크기를 지원한다.
            
        
        - 네트워크 계층( 인터넷 계층 )
            
            → 송, 수신 호스트 사이의 패킷 전달 경로를 선택 ( [[ROUTING]] ),
            
            라우팅 과정에서 생길 수 있는 트래픽 과다 현상을 제어하는 ==**‘혼잡 제어’**==,
            
            라우터 사이의 패킷 중개 과정에서 다루는 **==‘패킷 분할’==** 기능을 담당한다.
            
              
            
            → 네트워크 계층에서 전송 계층으로 제공하는 서비스는 크게 연결형 / 비 연결형으로
            
            나뉜다
            
            1. 연결형 서비스 : 상대적으로 신뢰성이 높은 서비스, 데이터를 전송하기 전에 연결을 미리 설정하는 송신 방식, 데이터가 모두 동일한 경로를 이용하기 때문에 목적지에 도착하는 순서가 송신 된 순서와 동일하다.
            
            1. 비 연결형 서비스 : 패킷 전달 순서, 분실 여부 등에서 연결형 서비스(TCP 등) 보다 신뢰성이 떨어지는 전송 방식. 이 방식을 이용하면 자체적으로 오류 제어와 흐름 제어 기능을 연결형 서비스보다 더 많이 수행해야 한다.
                
                또한, 패킷이 서로 다른 경로를 통해 목적지 호스트로 전달되기 때문에 패킷이 도착하는 순서를 보장하지 못한다. 따라서 상위 계층인 전송계층에서는 수신한 패킷 순서를 재조정하는 작업이 필요하다
                
                  
                
                ※ 네트워크 계층을 지원하는 IP 프로토콜은 비 연결형, 전송 계층의 TCP는
                
                연결형 서비스, 전송 계층의 다른 프로토콜인 UDP는 비 연결형 서비스를
                
                제공한다.
                
            
              
            
            → 인터넷 계층 프로토콜에는 IP 외에도 여러 규약이 있다.
            
            - Address Resolution Protocol ( ARP ) : 주소 변환 규약, 네트워크 계층 주소와 링크
                
                계층 주소 사이의 변환을 담당
                
            
            - Internet Group Management Protocol ( IGMP ) : 인터넷 그룹 관리 프로토콜,
                
                그룹 멤버십을 구성하거나, 그룹 관리를 위한 프로토콜
                
            
            - Internet Control Message Protocol ( ICMP ) : 인터넷 제어 메세지 프로토콜
                
                인터넷 통신 서비스 환경에서 오류에 대한 알림과 관련 된 메세지를 전달하는 목적
                
            
        
        - 데이터 링크 계층
            
            → X.25(?), 이더넷, 위성 통신
            
            → 데이터 전송의 최하위 계층으로, 네트워크 인터페이스 계층
            
            ( Network Interface Layer ) 으로 칭하기도 한다.
            
            → 물리적 전송 선로로 연결한 두 호스트 간의 전송 오류 문제를 다룬다.
            
            ( 프레임 전송 과정에서 프레임 분실, 변형과 같은 오류 발생을 검사하고, 복구한다 )
            
            → 이 계층은 데이터가 원하는 IP 주소에 도달하게 해주며,
            
            해당 네트워크 내의 연결 된 기기에 제대로 연결되어 있는지 확인하는 역할을 한다.
            
            → 원하는 기기의 MAC 주소를 확인하고 이더넷 케이블 및 와이파이를 통한
            
            데이터 전송을 관리하는 등의 작업을 담당한다.
            
        
    

  

- TCP IP 주소
    
    - 공개 IP 주소 , 비공개 IP 주소, 정적 IP 주소, 동적 IP 주소 등이 있다
    
    - TCP / IP 는 모든 유형의 주소에서 작동한다.
