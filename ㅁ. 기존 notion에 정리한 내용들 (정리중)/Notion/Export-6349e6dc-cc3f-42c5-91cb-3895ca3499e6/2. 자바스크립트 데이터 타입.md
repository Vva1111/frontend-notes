- ES6 기준, 자바스크립트는 7개의 데이터 타입(원시/객체) 을 제공한다.

||String|문자열|
||Boolean|논리적 참 / 거짓|

### Number

1. JavaScript에는 int, long, float, double 등과 같은 다양한 숫자 타입을 제공하지 않는다. 모든 값을 실수 값으로 처리한다.
    
    → 모든 값은 실수로 처리되기 때문에 정수로 표시되는 수 끼리 연산을 하더라도 실수가 나올 수 있다.
    

  

1. 또한, 2 진수, 8 진수, 16 진수 리터럴은 모두 64비트 부동 소수점 형식의 2 진수로 저장된다.

1. Infinity(양의 무한대), -Infinity(음의 무한대), NaN(산술 연산 불가) 을 표현할 수 있다.

  

### String

1. 원시 타입, 변경 불가능 한 값

### undefined

1. var 키워드로 선언한 변수는 암묵적으로 undefined로 초기화 된다.
    
    변수 선언에 의해 할당되어진 메모리 공간을 할당하기 전까지 빈 공간으로 두지 않고 자바스크립트 엔진에 의해 undefined로 초기화 된다.
    

1. 개발자가 의도한 값이 아닌, 자바스크립트 엔진에 의해 초기화 된 값.

1. 개발자가 의도적으로 undefined를 변수에 할당하는 작업은 권장되지 않음

1. 값이 없다는 것을 명시하기 위해서는 null 값을 할당

  

### null

1. 값이 없음을 의도적으로 명시하기 위해 사용

1. 변수에 null 값을 할당하는 것은 변수가 이전에 참조하던 값을 더 이상 참조하지 않겠다는 의미이며, 이전에 할당 되어 있던 값에 대한 참조를 명시적으로 제거하는 것을 의미한다.

1. 자바스크립트 엔진은 누구도 참조하지 않는 메모리 공간에 대해 가비지 콜렉션을 수행한다.

1. 다른 원시 값과는 다르게 Symbol() 함수를 통해 생성 가능하다.

### 데이터 타입의 필요성

- 자바스크립트는 메모리의 할당 및 해제를 위한 메모리 관리 기능을 언어 자원에서 담당하고 개발자의 직접적인 메모리 제어를 허용하지 않기 때문에 개발자가 명시적으로 메모리를 할당하고 해제할 수 없는  Unmanaged 프로그래밍 언어

- 자바스크립트 엔진은 데이터 타입(값의 종류)에 따라 정해진 크기의 메모리 공간을 확보한다. 즉, 변수의 타입에 따라 확보해야 할 메모리의 크기가 정해진다는 것이다.

- 메모리에 저장되어 있는 값을 참조하려면 한번에 읽어 들여야 할 메모리 공간의 크기(메모리 셀의 개수)를 알아야 한다. 값의 타입 단위에 따라 값을 읽지 않으면 데이터가 훼손된다.

- 예를 들어, 변수에 숫자 타입의 값이 할당되어 있다고 한다면, 자바스크립트 엔진은 해당 변수를 숫자 타입으로 인식하고, 숫자 타입은 8Byte 단위로 저장되므로 해당 변수를 참조했을 때 메모리 공간에 저장된 값을 8Byte 단위로 읽어 들인다.

- 아래 그림은 자바스크립트 엔진이 리터럴 100을 숫자 타입의 값으로 해석하고 저장하기 위해 8바이트의 메모리 공간을 확보한 후, 2진수로 저장하는 그림이다.

```jsx
var score = 100;
```

![[Untitled 35.png]]

- 메모리에 저장된 값은 데이터의 타입에 따라 다르게 해석될 수 있다.
    
    ( 0100 0001 을 숫자로 해석하면 65지만, 문자열로 해석하면 A가 된다.)
    
    그렇기 때문에, 변수에 할당된 타입을 참조해 메모리의 값을 어떻게 해석할지 결정한다.
    

  

→ 값을 저장할 때, 확보해야 하는 메모리 공간의 크기를 결정하기 위해

→ 값을 참조할 때 한번에 읽어 들여야 할 메모리 공간 크기를 결정하기 위해

→ 메모리에서 읽어 들인 2 진수를 어떻게 해석할지 결정하기 위해

데이터 타입이 필요하다.

  

### 동적 타이핑

1. **동적 타이핑이란?**
    
    - 자바스크립트는 변수 선언 시 자료형을 선언하지 않는 동적 언어 타입을 사용한다.
    
    - 변수에 어떤 데이터 값이라도 할당할 수 있으며, 할당하는 시점에 타입이 동적으로 결정된다. 또한 타입은 추후 변경 가능하다.
    

![[Untitled 36.png]]

![[Untitled 37.png]]

  

- foo 변수에 여러 타입을 설정할 수 있다.

  

  

- 즉, 자바스크립트는 선언이 아닌 할당에 의해 타입이 결정된다. 이를 ‘타입 추론’이라고 하며, 자바스크립트의 주요 특징 중 하나인 동적 타이핑이다.

- 변수는 타입을 갖지 않지만, 변수가 바라보는 ‘값’은 타입을 가지기 때문에 값에 의해 변수의 타입이 동적으로 결정된다.

- 이러한 동적 타이핑 방식을 사용하는 언어로는, Python, PHP, Ruby, List, Perl 등이 있다.

  

1. **동적 타이핑의 구조적 단점**
    
    - 복잡한 프로그램 내에서는 지속적으로 변화하는 값을 추적하기 어려울 수 있다.
    
    - 동적으로 변화하는 타입에 대해 변수의 값을 확인하기 전에는 타입을 확신할 수 없다.
    
    - **자바스크립트 엔진에 의해 개발자의 의도와는 상관 없는 타입 변환이 일어날 가능성이 있다.**
    
    - 유연성은 높지만, 신뢰성은 떨어진다는 큰 단점 발생.
    
    - 변수의 scope 제한, 전역 변수 사용 제한, 상수(const) 사용, 네이밍 규칙 강화 등을 통한 동적 타이핑 단점 보완 방법이 있다.
    

  

> [!important] 정적 타입 언어
> 
>   
> 
> - 반대 개념인 정적 타입 언어의 경우, 변수 선언 시 자료형을 사전에 설정해야 하며, 한 번 결정된 변수의 타입은 추후 변경할 수 없다. 또한, 컴파일 시점에 타입 체크를 수행하며, 이를 통과하지 못했을 때 에러와 함께 프로그램 실행을 막는다.
> 
> - C, C++, Java, kotlin, Go, Haskell, Rust, Scala 등의 정적 타입 언어들이 있다.
