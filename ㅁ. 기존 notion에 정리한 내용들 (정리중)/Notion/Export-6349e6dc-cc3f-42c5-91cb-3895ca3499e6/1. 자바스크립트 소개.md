### 표준화

1. 1996년, 마이크로소프트, 자바스크립트의 파생 버전인 Jscript를 IE에 탑재

1. JavsScript와 Jscript가 완전히 호환을 이루지 못하고, 넷스케이프와 마이크로소프느는 각각의 언어에 최적화 된 브라우저에서만 동작하는 기능을 경쟁적으로 추가하기 시작

1. 크로스 브라우징 이슈 발생

1. 96년 11월, 넷스케이프에서 컴퓨터 시스템 표준 관리 협회인 ECMA 인터네셔널에 자바스크립트의 표준화를 요청

1. 97년 7월, 자바스크립트 표준화 사양 초판 (ECMA 1) 완성

  

### 특징

1. 웹 브라우저에서 동작하는 유일한 프로그래밍 언어

1. 기본 문법은 C, 자바와 유사하며, self에서는 프로토타입 기반 상속, Scheme에서는 일급 함수의 개념을 차용했다.

1. 별도의 컴파일 작업을 수행하지 않는 인터프리터 언어
    
    1. 웹 문서의 구조를 동적으로 나타내기 위한 초기의 제작 컨셉에 적합
    
    1. 일반적으로 인터프리터 언어로 명명하긴 하지만, 인터프리터와 컴파일러의 장점을 결합한 형태
    
    1. 모던 브라우저에서 사용되는 인터프리터는 전통적인 컴파일러 언어처럼 명시적인 컴파일 단계를 거치지는 않지만 복잡한 과정을 거치며 일부 소스코드를 컴파일하고 실행한다.
    
    1. 인터프리터 언어의 장점인 동적 기능 지원과 컴파일 언어의 실행 속도가 느리다는 단점을 극복한다. (JIT)
    
    1. 하지만
        
        1. 런타임 내에서만 빠르게 컴파일 되며,
        
        1. 실행 파일이 생성되지 않고,
        
        1. 인터프리터의 도움 없이 실행할 수 없기 때문에
        
        컴파일 언어라고 할 수 없다.
        
    
    1. 프로토 타입 기반의 객체 지향 언어
    

  

### ECMAScript (ECMA-262)

1. 프로그래밍 언어의 값, 타입, 객체와 프로퍼티, 함수, 표준 빌트인 객체 등 핵심 문법 규정

1. 각 브라우저 제작사는 ECMA 사양을 준수해 브라우저에 내장되는 자바스크립트 엔진을 구현

1. 자바스크립트는 일반적으로 프로그래밍 언어의 기본 뼈대를 이루는 ECMAScript와 브라우저가 별도 지원하는 클라이언트 사이드 Web API ( DOM, BOM, Canvas, XMLHttpRequset, fetch, requestAnimationFrame, SVG, Web Storage, Web Component, Web Worker 등 )를 한번에 모두 아우르는 개념

1. 위의 클라이언트 Web API는 ‘월드 와이드 웹 콘소시엄’에서 별도의 사양으로 관리하고 있다. ([https://developer.mozilla.org/ko/docs/Web/API](https://developer.mozilla.org/ko/docs/Web/API))

1. 현재 ES14(2023. 06) 표준 사양 출현

  

### DOM(Document Object Model)

![[Untitled 27.png]]

1. HTML 문서의 계층적 구조와 정보를 표현해 이를 제어할 수 있는 API, 즉 프로퍼티와 메서드를 제공하는 트리 자료 구조

1. 브라우저의 렌더링 엔진은 HTML 문서를 파싱해 각 태그들을 하나의 객체로 만들어 브라우저가 이해할 수 있는 자료구조인 DOM 을 생성한다.

1. HTML과 자바스크립트를 연결해주는 역할로, 자바스크립트를 이용해 각 요소에 접근한다.

1. Document Node(HTML 문서 전체, DOM 트리 구조의 최상위 노드), Element Node(div, span 등의 태그), Attribute Node(input등의 태그 내부 속성), Text Node(문자), Comment Node(주석)

  

### BOM(Browser Object Model)

1. 자바스크립트가 브라우저와 소통하기 위해 만들어진 모델

1. 모든 개별 객체들은 최상위 객체인 window 아래에 존재한다.

1. 웹 페이지 내용을 제외한 웹 브라우저 창에 포함된 모든 객체 요소들을 의미한다.

1. window 속성에 속하며 document 문서가 아닌, window를 제어한다.

1. window, location(url 주소에 대한 정보 제공), document(현재 문서에 대한 정보), history(방문 기록 정보 제공), screen(브라우저의 외부 환경에 대한 정보 제공)

  

### Ajax( Asynchronous JavaScript And XML )

1. 99년, 자바스크립트를 이용해 브라우저와 서버 사이에 비동기 데이터 전송 방식(XMLHttpRequest) 등장

1. 페이지가 바뀔 때 마다 완전한 HTML 전체 코드를 서버로부터 웹 페이지 전체를 한번에 렌더링하는 방식에서 변경이 필요한 부분만 전송 받아 변경할 수 있는 방식으로 전환

1. 가장 유명한 예로, Ajax를 사용해 만든 구글 맵스의 지도 화면 전환 효과와 성능

  

### JQuery

1. 크로스브라우징 이슈를 해결하기 위해 만들어진 라이브러리 (2006년 개발)

1. DOM의 원활한 제어를 가능하게 함

  

### V8 JavaScript Engine

1. 자바스크립트를 이용한 웹 애플리케이션 구축 시도가 늘어나며 더욱 성능 좋은 브라우저 엔진의 필요

1. V8 엔진의 개발로 과거 웹서버에서 수행되던 적지 않은 로직들을 클라이언트로 이동할 수 있었고, 프론트엔드 개발 영역이 늘어남

1. V8 : C++로 개발되어 웹 브라우저를 만드는 데 기반을 제공하는 오픈소스 자바스크립트 엔진, 구글 크롬과 안드로이드 브라우저에 탑재되어 있다.

1. JIT(Just-In-Time compliation) 또는 동적 번역(프로그램을 실제 실행하는 시점에 기계어로 번역하는 컴파일 기법) 프로파일링을 통해 최적화 할 코드를 선별한 후 해당 코드들만 컴파일한다.

### Node.js

1. 2009년 발표된 구글 V8 자바스크립트 엔진으로 빌드 된 자바스크립트 런타임 환경

1. 브라우저의 자바스크립트 엔진에서만 동작하던 자바스크립트를 브라우저 이외의 환경에서도 동작할 수 있도록 자바스크립트 엔진을 브라우저에서 독립시킨 실행 환경

1. 서버 사이드 애플리케이션 개발에 주로 사용

1. 자바스크립트 언어를 사용

1. 비동기 I/O를 지원하며, 단일 스레드 이벤트 루프 기반으로 동작
