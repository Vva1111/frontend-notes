- 호스트 OS 상에 논리적인 구획(컨테이너)을 만들고, 애플리케이션을 작동시키기 위해 필요한 라이브러리나 애플리케이션을 하나로 모아서 마치 별도의 서버인 것 처럼 사용할 수 있게 만든 것

- 호스트 OS의 리소스를 논리적으로 분리시키고, 여러 개의 컨테이너가 리소스를 공유해 사용한다.

- 오버헤드가 적어 가볍고, 고속으로 작동할 수 있다.

하이퍼바이저 형 가상화 - 하드웨어 상에 가상화 전문 수행 소프트웨어(하이퍼바이저)를 배치해 하드웨어와 가상환경을 제어, 호스트 OS 없이 하드웨어를 직접 제어하기 때문에 자원을 효율적으로 사용할 수 있지만 가상환경 내에 별도의 OS가 작동하므로 가상 환경 시작에 걸리는 오버헤드가 커진다.

![[image 33.png]]

호스트 가상화 - 간편하게 가상 환경을 구축할 수 있으나, 호스트 OS 위에 게스트 OS를 두고 있기 때문에 오버헤드가 커진다.

  

- 보통 물리 서버에 설치한 호스트 OS의 경우, 그 위에서 움직이는 여러 애플리케이션은 똑같은 시스템 리소스를 사용하기 때문에 미들웨어나 라이브러리의 버전이 다른 경우, 각 애플리케이션이 서로 영향을 받지 않도록 주의해야 한다.

- 이에 반해, 컨테이너는 OS나 디렉토리, IP 주소 등과 같은 시스템 자원을 마치 애플리케이션이 각각 점유하고 있는 것처럼 보이게 할 수 있다.

- 컨테이너는 애플리케이션 실행에 필요한 모듈을 컨테이너 단위로 모을 수 있기 때문에 여러 개의 컨테이너를 조합해 하나의 애플리케이션을 구축하는 마이크로 서비스형 애플리케이션과 친화성이 높다.

- 컨테이너 기술과 가상화 기술은 비슷해 보이지만 목적이 다르다.
    
    - 컨테이너 기술은 애플리케이션의 실행 환경을 모아 이식성을 높이고 확장성이 좋은 환경에서의 작동을 지향
    
    - 가상화 기술은 서로 다른 환경을 어떻게 효율적으로 에뮬레이트하는지에 대해 고민
    

  

## 컨테이너의 역사

### FreeBSD Jail

- 오픈 소스 Unix인 FreeBSD의 기술

- FreeBSD 시스템을 Jail 이라고 불리는 독립된 작은 구획에 가둬 넣어 시스템을 분할할 수 있다.

- 특징으로는,
    
    - 프로세스의 구획화 : 똑같은 Jail에서 작동하는 프로세스만 엑세스할 수 있도록 프로세스를 분리
    
    - 네트워크의 구획화 : Jail 하나하나에 각각의 IP 주소가 할당되어 있으며, 여러 개의 주소를 할당할 수도 있다.
    
    - 파일 시스템의 구획화 : 조작할 수 있는 명령 혹은 파일 등을 제한할 수 있다.
    

### Solaris Containers

- Oracle의 상용 Unix인 Solaris에서 사용하는 컨테이너 기술

- Solaris 존
    
    - 하나의 OS 공간을 가상적으로 분할해 여러 OS가 작동하고 있는 것처럼 보여주는 소프트웨어 파티셔닝 기능
    
    - 베이스가 되는 OS 영역을 ‘global zone’, 구획화 된 가상 영역을 ‘non-global-zone’ 이라고 한다.
    
    - 각 non-global-zone 은 완전히 격리되어 있어 각각의 프로세스는 다른 non-global-zone에서 엑세스 할 수 없다.
    

- Solaris 리소스 매니저 기능
    
    - non-global-zone 에서 CPU 혹은 메모리와 같은 하드웨어 리소스를 배분하는 리소스 관리 기능
    
    - 중요도가 높은 업무 시스템에 대해 우선적으로 리소스를 할당할 수 있다.
    
    - Docker와 구조가 매우 비슷하다.
    

- Solaris Containers는 애플리케이션의 실행 환경을 존으로 구획화 해서 구획들을 제어한다
