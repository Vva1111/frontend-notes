- Dockerfile 안에서 사용할 수 있는 환경 변수나 컨테이너 안에서의 작업 디렉토리 지정

## **🌊 ENV 명령 (환경 변수 설정)**

> [!important]
> 
> `ENV <key> <value>`
> 
> `ENV <key>=<value>`

- 환경 변수를 설정하고 싶을 때 사용
    
    - 컨테이너 내부에서 `$MY_KEY`로 환경 변수 사용 가능
    

- 명령별로 Docker 이미지 생성

- `docker container run` 실행 시 `-e` 혹은 `--env` 옵션으로 실행 시 환경 변수 변경 가능
    
    ```bash
    docker container run -e <key>=<new value>
    docker container run -env <key>=<new value>
    ```
    

  

### ✔ 환경 변수를 지정하는 두 가지 방법

**1️⃣ key value 형**

- 단일 환경 변수에 하나의 값을 설정

- 첫 번째 공백을 기준으로 key와 value를 구분

```docker
# 3개의 ENV 명령
# 3개의 이미지 생성
ENV myName "Shiho ASA"
ENV myOrder Gin Whisky Calvados
ENV myNickName miya
```

  

**2️⃣ key=value 형**

- 한 번에 여러 개의 값을 설정

- ENV 문법 상, 공백을 기준으로 처리되기 때문에 띄어쓰기가 포함된 문자열은 `\`이스케이프 처리 혹은 따옴표 처리 필요

```docker
# 1개의 ENV 명령
# 1개의 이미지 생성
ENV myName="Shiho ASA" \
		myOrder=Gin\ Whisky\ Calvados \
		myNickName=miya
```

## **🌊 WORKDIR 명령 (작업 디렉토리 설정)**

> [!important]
> 
> `WORKDIR <작업 디렉토리 경로>`

- RUN, CMD, ENTRYPOINT, ADD, COPY가 실행될 기본 디렉토리를 설정

- 지정한 디렉토리가 없을 경우 생성함

- Dockerfile 내에서 여러 번 사용 가능
    
    - 상대 경로라면 이전 WORKDIR 명령의 경로에 대한 상대 경로가 됨
    

- 환경 변수 사용 가능

  

```docker
WORKDIR /first
# /first

WORKDIR second
# /first/second

WORKDIR /third
# /third

WORKDIR fourth
# /third/fourth
```

## **🌊 USER 명령 (사용자 지정)**

> [!important]
> 
> `USER <사용자명/UID>`

- RUN, CMD, ENTRYPOINT를 실행할 사용자를 지정

- USER 명령에서 지정하는 사용자는 RUN 명령으로 미리 작성해야 함

  

```docker
RUN ["adduser", "asa"]
RUN ["whoami"]
# root
USER asa
RUN ["whoami"]
# asa
```

- `whoami` : 사용자명을 표기하기 위한 Linux 명령

- `USER asa` 명령 후의 명령들은 `asa`로 실행됨

## **🌊 LABEL 명령 (라벨 지정)**

> [!important]
> 
> `LABEL <키 명>=<값>`

- 이미지에 정보 제공

- `\`를 사용하여 하나의 명령으로 처리 가능 (권장)

  

```docker
# LABEL maintainer "Shiho ASA<asashiho@mail.asa.seoul>"
# 위 명령은 도커 버전에 따라 일부 실행될 수도 있으나 비문법적 표현임.
# key=value 형태로 사용할 것.

LABEL maintainer="Shiho ASA<asashiho@mail.asa.seoul>"
LABEL title="WebAP"
LABEL version="1.0"
```

  

> [!important]
> 
> **MAINTAINER 명령 (deprecated)**
> 
> ---
> 
> 구 버전(1.13 이전)에는 Dockerfile 작성자를 MAINTAINER 명령으로 기술했으나, 현재 권장하지 않는다. LABEL과 달리 여러 정보를 추가할 수 없고, 이미지의 정보를 한번에 관리하기 어려워 deprecated 되었다.
> 
> MAINTAINER 명령을 사용하면 LABEL과 달리 inspect로 전체를 확인해야 하며, `Author` 영역에 별도 표기된다.

## **🌊 EXPOSE 명령 (포트 지정)**

> [!important]
> 
> `EXPOSE <포트 번호>`

- 컨테이너의 공개 포트 번호 지정

- Docker에게 실행 중인 컨테이너가 listen하고 있는 네트워크를 알려줌

- 컨테이너가 어떤 포트를 사용할 것이라고 알림 (실제 개방 아님)
    
    - `docker container run` 의 `-p` 옵션을 사용해 어떤 포트를 호스트에 공개할지 정의
        
        = `docker container run -p <호스트 포트 번호>:<컨테이너(내부) 포트 번호>`으로 실행해야 개방됨
        
    

  

```docker
EXPOSE 8080
```

## **🌊 ARG 명령 (Dockerfile 내 변수 설정)**

> [!important]
> 
> `ARG <이름>[=기본값]`

- Dockerfile 안에서 사용할 변수 정의

- 변수의 값에 따라 생성되는 이미지 내용이 변경됨
    
    - Dockerfile 빌드 시 `--build-arg` 옵션으로 값 지정 가능
    
    - `--build-arg` 사용하지 않을 경우 ARG 기본값 전개
        
        - ARG 기본값이 없을 경우 해당 변수는 빈값(`""`)으로 처리됨
        
        - 오류가 발생하지 않음!
        
    

- ENV와 달리 Dockerfile 내에서만 사용 가능

  

```docker
# 변수 정의

ARG YOURNAME="asa"
RUN echo $YOURNAME
```

```bash
docker build . --build-arg YOURNAME=shiho
```

## **🌊 SHELL 명령 (기본 쉘 설정)**

> [!important]
> 
> `SHELL ["쉘의 경로", "파라미터"]`

- 쉘 형식으로 명령을 실행할 때의 기본 쉘 설정
    
    - RUN, CMD, ENTRYPOINT
    

- Linux의 기본 설정된 쉘은 `["/bin/sh", "c"]`, Windows는 `["cmd", "/S", "/C"]` 로 설정되어 있음

  

```docker
# 기본 쉘 지정
SHELL ["/bin/bash", "-c"]

# RUN 명령 실행
RUN echo hello
```
