Docker Network 구조

- 컨테이너를 생성하면, 컨테이너는 호스트와 통신하기 위한 eth0 라는 네트워크 인터페이스를 할당받는다.

- 이 때, 동시에 호스트에도 veth(virtual ethernet) 라는 네트워크 인터페이스가 할당되고, 컨테이너에 할당된 eth0 인터페이스와 통신하게 된다.

- 호스트에 할당된 veth 인터페이스는 docker0와 바인딩되고, docker0는 호스트의 eth0 인터페이스와 연결되어 외부로부터 들어온 요청을 처리한다.

  

### docker0

- 호스트의 eth0 네트워크 인터페이스와 직접적으로 연결되는 네트워크 인터페이스

- 도커를 설치하게 되면 도커 내부 로직에 의해 호스트는 자동으로 IP를 할당 받게 되는데, 이는 172.17.x.x 로 시작하며, netmask는 255.255.0.0 으로 설정된다.

![[image 38.png]]

ifconfig: 서버의 IP 주소 정보 제공 및 수정 기능 제공

- docker0 인터페이스는 호스트의 eth0 인터페이스와 컨테이너의 eth0 인터페이스 사이 중재자 역할을 하는 가상의 브릿지 네트워크

- 컨테이너 생성 - 컨테이너 내부에 할당되는 eth0와 호스트에 할당되는 veth가 연결 - docker0에 veth가 바인딩

- 도커를 설치하면 기본적으로 생성되는 브릿지 네트워크의 정보에서 bridge.name이 docker0로 되어있는 것을 확인할 수 있다. 또한 docker0의 IP와 기본 브릿지 네트워크의 Gateway IP가 일치하는데, 이는 기본 브릿지 네트워크가 Gateway로 호스트에 제공된 docker0 인터페이스를 사용한다는 사실을 알 수 있다.

![[image 39.png]]

- 따라서, 컨테이너가 기본 브릿지 네트워크를 사용하는 경우 외부로 통신할 때는 무조건 docker0 인터페이스를 거쳐야 한다.

- 도커는 호스트의 iptables를 통해 외부에서 들어오는 요청을 관리하는데, 컨테이너가 활성화 되어있을 경우, iptables를 확인하면 ‘설정된 포트로 요청이 들어오는 경우 docker0 인터페이스를 거쳐 해당 주소로 요청을 전달해라’라는 규칙이 추가된다.

- 포트포워딩이 활성화된 컨테이너를 생성하면 iptables에 규칙을 추가하고, 해당 규칙을 통해 외부에서 들어오는 요청을 처리한다.

  

### veth

- 컨테이너를 실행하면, 호스트에 veth 인터페이스가 할당된다.

- veth(virtual ethernet) 인터페이스는 가상의 네트워크 인터페이스로, 자신과 연결된 컨테이너의 네트워크 인터페이스와 패킷을 주고받는 식으로 동작한다.

- 도커에 의해 자동 생성된다.

- veth 인터페이스는 항상 쌍으로 생성된다.
    
    - 하나는 vethxxxx 라는 이름으로 호스트에 생성되며, docker0에 바인딩 되는 형식이고, 하나는 컨테이너 내부에 eth0 라는 이름으로 생성되어 veth 인터페이스와 연결되는 방식이다.
    

- 호스트에 실행 중인 컨테이너가 하나도 없을 경우, docker0에 veth가 바인딩 되지 않는다.

  

- 기존에 항상 백그라운드에서 항상 실행되고 있는 gitlab-runner 컨테이너가 있기 때문에 btctl(Linux 커널에서 이더넷 브릿지 구성을 설정, 유지 및 관리하는 도구)로 확인했을 때, veth 하나가 docker0에 바인딩 되어 있는 것을 확인할 수 있다.
