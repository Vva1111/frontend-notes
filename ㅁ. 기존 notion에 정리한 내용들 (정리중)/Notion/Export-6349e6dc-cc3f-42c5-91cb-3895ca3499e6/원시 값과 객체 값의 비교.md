  

### 원시 값

- 변경 불가능, 읽기 전용 값

- 원시 값을 변수에 할당할 경우, 변수(확보된 메모리 공간)에는 실제 값이 저장된다.

- 원시 값을 갖는 변수를 다른 변수에 할당하면, 원본의 원시 값이 복사 되어 저장된다. (값에 의한 전달)

![[Untitled 304.png]]

- 원시 값을 할당한 변수에 새로운 원시 값을 재 할당할 경우, 새로운 메모리 공간을 확보한 후, 변수는 새롭게 재 할당한 원시 값을 가리킨다. 이 때, 변수가 참조하던 메모리 공간의 주소가 바뀐다.

- 원시 값이 변경 가능하다면, 예기치 않은 값의 변경을 초래할 수도 있고, 이는 상태의 변경을 추적하기 어렵게 만든다.

- 문자열은 몇 개의 문자로 이루어졌는지에 따라 필요한 메모리 공간의 크기가 결정된다.

- 자바스크립트의 문자열은 원시 타입이며, 변경 불가능하다.

- 문자열은 유사 배열 객체이므로, 마치 배열처럼 프로퍼티 값에 접근 가능하며, length 프로퍼티를 갖는다. 또한, for문으로 순회 가능하다.

- 하지만 불변한 값이기 때문에 프로퍼티를 이용한 값 변경은 불가능하다.

  

### 값에 의한 전달

- 변수에 변수를 할당했을 때 무엇이 전달되는지 파악

![[Untitled 305.png]]

![[Untitled 306.png]]

  

- 변수에 원시 값을 갖는 변수를 할당하면 할당 받는 변수(copy) 에는 할당 되는 변수(score)의 원시 값이 복사 되어 전달된다. 이를 ‘값에 의한 전달’ 이라고 한다.

![[Untitled 307.png]]

- score와 copy는 80을 가지고 있다는 점에서는 동일하지만, 이 두 값 80은 서로 다른 메모리 공간에 저장된 별개의 값이다.

- ECMAScript는 변수와 메모리 관리에 대한 명확한 정의를 해 놓지 않은 상태이기 때문에 실제 동작 방식은 차이가 있을 수 있으며, ‘값에 의한 전달’ 이라는 표현 또한 오해의 소지가 있다.

- ‘값에 의한 전달’ 표현의 경우, 자바스크립트 환경 내에서는 오해의 소지가 있다. 변수와 같은 식별자에는 값이 아니라 해당 값을 가리키는 메모리 주소를 기억하고 있기 때문이다.

- 즉, 식별자는 메모리 주소에 붙인 이름이라고 볼 수 있으며, 해당 메모리 주소를 통해 메모리 공간에 저장된 값에 접근할 수 있는 것이다.

- 중요한 점은, 두 변수의 값은 서로 다른 메모리 공간에 저장된 별개의 값이 되어 어느 한 쪽에서 재 할당을 통해 값을 변경하더라도 서로 간섭할 수 없다.

- 원시 값을 갖는 변수의 값을 변경하려면 재 할당을 통해 메모리에 원시 값을 새롭게 생성해야 한다.

  

### 객체 값

- 객체 값 : 객체(참조) 타입의 값, 변경 가능한 값

- 프로퍼티의 수가 정해져 있지 않고, 동적으로 추가, 삭제될 수 있기 때문에 확보해야 할 메모리의 크기를 사전에 정해둘 수 없다.

- 객체는 자료 구조나 관리 방식이 복합적이기 때문에 브라우저 별 엔진마다 구현 방식이 다를 수 있다.

- 크기가 매우 클 수도 있으며, 프로퍼티에 접근하는 일도 비용이 많이 들기 때문에 원시 값과는 다른 방식으로 동작하도록 설계되어 있다.

- 객체를 할당한 변수가 기억하는 메모리 주소를 통해 메모리 공간에 접근하면 ‘참조 값’에 접근할 수 있다. 참조 값은 생성된 객체가 저장된 메모리 공간의 주소. 그 자체이다.

![[Untitled 308.png]]

```jsx
var person = {
  name : 'Lee'
};

// person 변수는 실제 객체의 값을 참조할 수 있는 메모리 주소만 가지고 있다.
// 해당 참조 값을 통해 실제 객체 값에 접근할 수 있다.
```

- 객체를 할당한 변수를 참조하면 메모리에 저장되어 있는 참조 값을 통해 실제 객체에 접근한다.

- 객체는 변경 가능한 값이기 때문에 메모리에 저장된 값을 직접 수정할 수 있다. 객체 값을 변경한다 해도 재 할당이 필요하지 않기 때문에 객체를 할당한 변수의 참조 값은 변경되지 않는다.

- 메모리 사용의 효율과 성능을 위해 어느 정도의 구조적 단점을 감안한 설계

  

### 객체의 구조적 단점

- 객체의 구조적인 단점으로, 원시 값과는 다르게 여러 개의 식별자가 하나의 객체를 공유할 수 있다는 점이 있다.

- 객체를 가리키는 원본 변수를 다른 변수에 할당하면 원본의 참조 값(변수가 가리키는 값)이 복사 되어 전달된다.(참조에 의한 전달)

![[Untitled 309.png]]

- 이 때, 원본 Person과 사본 Copy는 저장된 메모리 주소는 다르지만 동일한 참조 값을 가진다. 모두 동일한 객체를 가리킨다는 뜻이다.(두 식별자가 하나의 객체를 공유)

- 따라서, 원본 또는 사본 중 하나의 객체를 변경하게 되면 서로 영향을 주고 받을 수 있다.

> [!important] ‘값에 의한 전달’과 ‘참조에 의한 전달’은 식별자가 기억하는 메모리 공간에 저장되어 있는 값을 복사해서 전달한다는 면에서 동일하다. 변수에 저장되어 있는 값이 원시 값인지, 객체 값인지의 차이만 있을 뿐이다. 따라서, 자바스크립트에는 ‘참조에 의한 전달’은 존재하지 않는다고 보는 것이 정확하다고 할 수 있다.(포인터가 존재하는 언어에서의 뜻과는 상이할 수 있으며, 이해를 돕기 위한 단어 선택)

> [!important] 두 식별자가 하나의 객체를 참조하게 되면 복사된 변수에 할당된 객체 값을 조정하면 원본 객체의 값도 함께 영향을 받는 문제가 생긴다. 이를 위해 'deep copy' 라는 방식으로 새로운 변수에 새로운 객체의 레퍼런스를 할당해야 한다.
