## **🌊 Docker 이미지 빌드**

Docker 명령으로 빌드할 수 있지만, Cloud Container Builder를 사용하면 Dockerfile의 빌드부터 ~~Cloud Source Repositories~~ Github의 업로드까지 하나의 명령으로 모아 실행할 수 있음

  

> [!important]
> 
> GCP에서 아래 API 라이브러리 설치
> 
> - **Kubernetes Engine API**
> 
> - **Google Container Registry API**
> 
> - **Cloud Build API**

  

### ▶️ Docker 이미지 빌드 파일

- Container Builder를 사용하여 Docker 이미지를 작성할 때 송신될 빌드 리퀘스트를 정의한 파일

- JSON 또는 YAML 형식으로 기술

- 빌드 리퀘스트 내용
    
    - **빌드 스텝**
        
        - 컨테이너를 빌드하기 위해 실행할 명령
        
        - 빌드가 완료되면 Google Container Registry에 Docker 이미지가 업로드(Push) 됨
        
    
    - **이미지**
        
        - 생성할 Docker 이미지 정보를 설정
        
    

  

### ▶️ 샘플 이미지 빌드 파일 구성 설명 (config/cloudbuild.yaml)

```yaml
steps:
  - name: 'gcr.io/cloud-builders/docker'
    args: ['build', '-t', 'gcr.io/$PROJECT_ID/imageview:blue', './blue']
  - name: 'gcr.io/cloud-builders/docker'
    args: ['build', '-t', 'gcr.io/$PROJECT_ID/imageview:green', './green']
images: ['gcr.io/$PROJECT_ID/imageview:blue', 'gcr.io/$PROJECT_ID/imageview:green']
```

- **[steps] 필드**
    
    - 빌드 단계 정의
    
    - [name]: docker build 명령어를 실행할 컨테이너 지정
        
        - `gcr.io/cloud-builders/docker` : Google Cloud에서 제공하는 Docker CLI 이미지
        
    
    - [args]: 실행할 명령어의 인수 정의
        
        - `-t gcr.io/$PROJECT_ID/imageview:blue`: 업로드할 이미지 이름과 태그 설정
        
        - `./blue`: Dockerfile이 있는 디렉토리
        
    
    - 즉, 여기서는 blue와 green 버전의 Docker 이미지를 각각 빌드
    

- **[images] 필드**
    
    - 빌드로 생성된 이미지 중 업로드할 이미지 목록 설정
    
    - 여기서 지정된 이미지가 GCR에 자동으로 푸시됨
    

  

> [!important]
> 
> `docker build -t`와 `images:` 필드에서 이미지 이름 지정의 차이점  
> **1️⃣** `docker build -t`에서 이미지 이름과 태그 지정
> 
> 📌 `**-t**` 옵션을 사용하여 Docker 이미지의 이름과 태그를 설정
> 
> ```yaml
> args: ['build', '-t', 'gcr.io/$PROJECT_ID/imageview:blue', './blue']
> ```
> 
> ✅ `docker build -t`를 실행하는 단계에서 이미지의 이름과 태그가 지정됨!
> 
>   
> 
> ---
> 
> **2️⃣** `images:` 필드에서 하는 역할
> 
> Cloud Build가 빌드된 이미지를 GCP의 Container Registry에 푸시하도록 지정
> 
> ```yaml
> images: ['gcr.io/$PROJECT_ID/imageview:blue', 'gcr.io/$PROJECT_ID/imageview:green']
> ```
> 
> - Cloud Build는 `images:` 필드에 지정된 이미지를 자동으로 GCR에 업로드함
> 
> - `docker build -t`로 만든 이미지가 여기에 있는 리스트와 일치해야 함.
>     
>     - 만약 `images:`에 등록되지 않으면, 빌드된 이미지는 자동 푸시되지 않음
>     
> 
>   
> 
> ✅ `images:` 필드는 Cloud Build가 빌드된 이미지를 GCR에 업로드하는 역할

  

- 빌드 실행

    cd ~/dockertext2/chap09

### ▶️ Cloud Container Builder 트리거

- → 저는 찾아서 해보다가 저장소 연결이 안돼서 때려침… 되신 분 제보 부탁…
    
    - 트리거를 작성함으로써 변경이 생기는 타이밍에 Docker 이미지를 빌드할 수 있음
    
      
    
    - GCP Cloud Build와 연동 (확인)
        
        **1️⃣ GCP Cloud Build 활성화**
        
        - 트리거 사용 설정([링크](https://cloud.google.com/build/docs/automating-builds/create-manage-triggers?hl=ko))
        
        - Cloud Build 서비스 계정에 권한 추가
            
            ```bash
            gcloud projects add-iam-policy-binding $PROJECT_ID \
              --member=serviceAccount:$PROJECT_ID@cloudbuild.gserviceaccount.com \
              --role=roles/storage.admin
            ```
            
        
          
        
        **2️⃣ GCP에서 Cloud Build 트리거 설정**
        
        1. Cloud Build 트리거 페이지로 이동
        
        1. 트리거 추가
            
            1. 이름: docker-build
            
        
        1. GitHub 연결
            
            - `GitHub 저장소 연결` 클릭
            
            - `dockertext2` 리포지토리 선택
            
        
        1. 트리거 설정
            
            - **이벤트:** `main` 브랜치에 푸시(push)될 때 실행
            
            - **빌드 구성 파일:** `chap09/config/cloudbuild.yaml`
            
        
          
        
        📌 **GCP에서 Cloud Build 트리거를 수동으로 실행하는 방법 (테스트)**
        
        ```bash
        gcloud builds triggers run <트리거_이름> --branch=main
        ```
        
        - Cloud Build에서 설정한 트리거의 이름을 넣어야 함
            
            ```bash
            gcloud builds triggers list
            ```
            
            위 명령어로 트리거 목록 확인 가능!
            
        
          
        
        **3️⃣ GCP에서 빌드 실행 확인**
        
        - 빌드 상태 확인
            
            ```bash
            gcloud builds list
            ```
            
        
        - 빌드 로그 확인
            
            ```bash
            gcloud builds describe <빌드_ID>
            ```
