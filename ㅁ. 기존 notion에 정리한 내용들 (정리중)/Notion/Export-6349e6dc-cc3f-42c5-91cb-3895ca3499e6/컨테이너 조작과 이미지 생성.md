# 4.4 가동 중인 Docker 컨테이너 조작

## 가동 컨테이너 연결(docker container attach)

```docker
docker container attach [컨테이너 ID 또는 이름]

# /bin/bash가 실행되고있는 sample 컨테이너에 연결하기
docker container attach sample
```

`docker container attach` 명령을 사용하면 실행 중인 컨테이너에 "직접 연결"된다. 이때 연결된 컨테이너 내에서 실행되는 프로세스(예: `/bin/bash` 셸)와 상호작용할 수 있다. (컨테이너 내부의 터미널처럼 동작)

![[image 63.png]]

sample 만들기

![[image 64.png]]

→ 연결된 상태에서 명령어를 실행하거나 컨테이너의 출력을 실시간으로 볼 수 있지만, 하나의 터미널 세션에 묶여있게 된다.

- 컨테이너에 연결한 후 종료 `Ctrl + C` (컨테이너가 종료될 수 있다.)

- 컨테이너를 종료하지 않고 단순히 세션 분리 `Ctrl + P` 를 누른 후, `Ctrl +Q`
    
    - 터미널 연결을 끊는 역할로, 컨테이너는 계속 백그라운드에서 실행되고 실제로 종료되지 않는다.
    

  

  

## 가동 컨테이너에서 프로세스 실행(docker container exec)

가동 중인 컨테이너에서 새로운 프로세스를 실행 할 때 사용한다.

가동중이지 않은 컨테이너는 docker container start를 먼저 해야한다.

인터랙티브 모드나 백그라운드 실행을 통해 다양한 작업을 할 수 있으며, 주로 디버깅이나 관리 작업에 사용된다.

```docker
docker container exec [옵션] <컨테이너 식별자> <실행할 명령> [인수]
```

![[image 65.png]]

- 가동중인 컨테이너 확인  
- 가동중인 컨테이너 내 프로세스 실행  
- 가동중인 컨테이너 내 명령 직접 실행

|옵션|설명|
|---|---|
|`--detach` , `-d`|명령을 백그라운드에서 실행|
|`--tty` , `-t`|tty 단말 디바이스를 사용|
|`--user` , `-u`|사용자명 지정|
|`--interactive` , `-i`|컨테이너의 표준 입력을 연다|

  

  

## 가동 컨테이너의 프로세스 확인(docker container top)

가동 중인 컨테이너에서 실행되고 있는 프로세스 확인

→ 컨테이너 모니터링 및 디버깅,트러블 슈팅 등에 사용한다.

```docker
docker container top <컨테이너명>
# docker container top webserver
```

  

  

## 가동 컨테이너의 포트 전송 확인(docker container port)

가동 중인 컨테이너에서 실행되고 있는 프로세스가 전송되고 있는 포트 확인

- _==well-known 포트를 피해 보통 애플리케이션에서는 1024 이후의 포트를 사용한다!==_

```docker

# docker container port mynginx
```

# 4.5 Docker 이미지 생성

컨테이너는 이미지를 바탕으로 작성하지만, 컨테이너를 바탕으로 이미지를 작성할 수도 있다.

## 컨테이너로부터 이미지 작성(docker container commit)

```docker
docker container commit [옵션] <컨테이너 식별자> [이미지명[:태그명]]
```

|옵션|설명|
|---|---|
|`--author` , `-a`|작성자 지정|
|`--message` , `-m`|메시지 지정|
|`--change` , `-c`|commit 시 Dockerfile 명령 지정|
|`--pause` , `-p`|컨테이너 일시 정지 후 commit|

## 컨테이너를 tar 파일로 출력(docker container export)

가동중인 컨테이너의 디렉토리/파일을 모아 tar 파일을 만들 수 있다.

tar 파일을 바탕으로 다른 서버 에서 컨테이너를 가동시킬 수 있다.

## 이미지 저장(docker image save)

이미지를 tar 파일로 저장

```docker
docker image save [옵션] <저장 파일명> [이미지명]
```

![[image 76.png]]

yepark/test:1.1 이미지 → export.tar로 저장

|   |   |
|---|---|
|**옵션**|**설명**|
|`-o, --output`|저장할 파일의 경로와 파일 이름을 지정  <br>지정하지 않으면 출력이 표준 출력  <br>예: `-o myimage.tar`|
|`--help`|`docker image save` 명령어에 대한 도움말을 표시|
|`-i, --input`|이미지를 tar 파일로 저장할 때, 입력으로 사용할 이미지 이름을 지정|
|`--quiet`|이미지의 저장 상태나 진행 상황을 표시하지 않음|

  

  

## 이미지 읽어 들이기(docker image load)

tar 파일로부터 이미지 로드

```docker
docker image load [옵션]
```

![[image 77.png]]

export.tar 파일에서 이미지 읽기

|   |   |
|---|---|
|**옵션**|**설명**|
|`-i, --input`|`tar` 파일을 지정하여 이미지를 로드  <br>예: `-i myimage.tar`|

> [!important]
> 
> ### **export/import 와 save/load 차이?**
> 
> - `export` / `import`:
>     
>     - `export`는 컨테이너의 파일 시스템만 내보낸다.  
>         이미지는 포함되지 않으며, 이 방식으로 내보낸 `tar` 파일은 복원 시 Docker 이미지로 변환되지 않는다.
>     
>     - `import`는 외부에서 가져온 `tar` 파일을 기반으로 새로운 Docker 이미지를 생성한다.  
>         이미지 내의 모든 메타데이터를 포함하지 않기 때문에 태그나 레이어 정보는 수동으로 설정해야 한다.
>     
> 
> - `save` / `load`:
>     
>     - `save`는 Docker 이미지를 `tar` 파일로 저장하며, 이미지에 포함된 모든 레이어와 메타데이터도 함께 저장한다.
>     
>     - `load`는 `tar` 파일에서 이미지를 로드하며, 이미지와 관련된 모든 정보(레이어, 태그 등)가 복원된다.
>     
> 
>   
> 
> `export`/`import`는 컨테이너의 파일 시스템을 기반으로 작업하는 반면, `save`/`load`는 Docker 이미지 전체를 저장하거나 불러오는 데 사용된다.

  

## 불필요한 이미지/컨테이너를 일괄 삭제(docker system prune)

사용하지 않는 이미지, 컨테이너, 볼륨, 네트워크 일괄 삭제

```docker
docker system prune [옵션]
```

|   |   |
|---|---|
|**옵션**|**설명**|
|`-a`, `--all`|사용되지 않는 모든 이미지(중간 레이어 포함)를 제거  <br>기본적으로는 dangling(미사용) 이미지만 삭제|
|`-f`, `--force`|확인 메시지를 생략하고 바로 삭제를 실행|
|`--volumes`|사용되지 않는 볼륨도 함께 삭제  <br>(기본적으로는 볼륨은 삭제되지 않음)|
|`--filter`|특정 조건에 맞는 리소스만 삭제하도록 필터링|
